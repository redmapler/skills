## 1. A+ 内容基础规范

### 1.1 什么是 A+ 内容

A+ 内容（A+ Content / Enhanced Brand Content）是亚马逊品牌注册卖家（Brand Registered sellers）可使用的高级图文描述模块，位于 Listing 详情页的产品描述区域。与普通文字描述相比，A+ 内容允许卖家用宽幅图片 + 图文排版讲述品牌故事、展示核心卖点、对比规格参数，是提升转化率的关键视觉资产。

> 注意：A+ 内容仅对完成品牌注册的卖家开放。未注册品牌的 Listing 无法使用 A+ 内容，本文件所有内容均假设卖家已完成品牌注册。

### 1.2 A+ 内容的两种版式

A+ 内容图片分为 Desktop（桌面版）与 Mobile（移动版）两种版式。由于 PC 端与手机端的浏览场景差异巨大，两种版式必须独立构图，**禁止直接缩放复用**。

| 维度 | Desktop（桌面版） | Mobile（移动版） |
|------|-------------------|-----------------|
| 尺寸 | 1464 × 600 px 或 1940 × 1200 px | 1600 × 1200 px |
| 比例 | 横版 ~2.44:1 或 ~1.62:1 | 竖版 4:3 |
| 构图原则 | 宽幅横图，左右分区，信息密度可较高 | 必须独立构图，不可缩放桌面版；文字更大，信息精简 |
| 文字字号 | 至少 36px | 至少 48px（小屏可读性要求更高） |
| 浏览场景 | PC 端横屏浏览 | 手机竖屏滑动浏览 |

> 重要：以上为项目默认尺寸，桌面版支持 1464 × 600（宽幅横图）和 1940 × 1200（大尺寸横图）两种规格，可按产品类型和视觉需求选择。实际上传前以当前 Seller Central 的 A+ 模块要求为准。亚马逊可能调整不同 module 的尺寸要求。

### 1.3 A+ 内容与 Listing 图的核心区别

| 维度 | Listing 主图/副图 | A+ 内容图片 |
|------|-------------------|-------------|
| 出现位置 | Listing 详情页主图区 | 详情页产品描述区 |
| 默认尺寸 | 1600 × 1600（方形） | Desktop 1464 × 600 或 1940 × 1200 / Mobile 1600 × 1200 |
| 背景要求 | 主图须纯白 | 可使用场景背景、色块、渐变 |
| 文字标注 | 主图禁止文字，副图可少量标注 | 鼓励图文排版，文字为主卖点载体 |
| 资产数量 | 默认 1 主图 + 副图（最多 9 张） | Desktop 5 张 + Mobile 5 张（标准规划） |
| 文件命名 | `listing-01.jpg` ~ `listing-09.jpg` | `aplus-desktop-01.jpg` / `aplus-mobile-01.jpg` |

### 1.4 A+ 图片命名规范

| 类型 | 命名规则 | 说明 |
|------|----------|------|
| A+ 桌面版 | `aplus-desktop-01.jpg` ~ `aplus-desktop-05.jpg` | 对应 5 张桌面版 |
| A+ 移动版 | `aplus-mobile-01.jpg` ~ `aplus-mobile-05.jpg` | 对应 5 张移动版 |

> 命名规则与 `SKILL.md` 的 Output Contract 一致，所有输出写入 `/workspace/outputs/[产品ID]/`。

## 2. 标准 5 张 A+ 图规划模板

> 以下为标准 A+ 规划：Desktop 5 张 + Mobile 5 张，共 10 张。
> Desktop 与 Mobile 主题一一对应，但构图独立设计。
> 产品不可变层与产品锁段落全部复用 Listing 阶段已锁定的内容，此处只描述 A+ 专属的视觉表达。

### 2.1 五张图总览

| 序号 | 主题 | Desktop 画面内容 | Mobile 画面内容 |
|------|------|-----------------|----------------|
| 1 | 品牌横幅 / Hero | 产品在生活场景中的宽幅横图，左侧产品+右侧场景，品牌色块过渡 | 产品居中大图，上方短标题，下方场景背景 |
| 2 | 核心卖点 1 | 左侧产品特写+右侧图标/文字说明，横向并列 | 上方产品特写+下方文字说明，纵向排列 |
| 3 | 核心卖点 2 | 同上结构，换一个卖点 | 同上结构，文字更精简 |
| 4 | 规格对比 / 信息图 | 横向对比表或规格信息图卡，多列布局 | 纵向信息列表，每行一个规格，大字号 |
| 5 | 使用场景 / 套装内容 | 宽幅场景图或套装平铺，左右展示 | 场景图或套装平铺，纵向居中展示 |

### 2.2 逐张详细说明

#### 图 1：品牌横幅 / Hero

| 版式 | 详细画面内容与构图说明 |
|------|----------------------|
| **Desktop** | 宽幅横图（1464×600）。左侧约 40% 区域放置产品主体（产品占满左侧高度，品牌色块作为背景过渡），右侧约 60% 区域展示产品所处的真实生活场景（如窗帘挂在窗前、电器放在台面）。左右之间用品牌主色渐变色块自然过渡，避免硬边。顶部或左下角放置品牌 Logo 与一句短标语（≤3 词）。强调横向叙事感，视觉重心从左到右流动。 |
| **Mobile** | 竖版图（1600×1200）。产品居中放大占据画面中上部约 55% 高度，产品下方为生活场景背景（虚化处理突出产品）。画面顶部留白区放置品牌短标题（大号粗体，≥48px）。底部可放一句极短标语。产品必须独立居中构图，不得裁切桌面版横向画面。 |

#### 图 2：核心卖点 1

| 版式 | 详细画面内容与构图说明 |
|------|----------------------|
| **Desktop** | 横向左右分区。左侧约 50% 为产品局部特写（聚焦体现该卖点的部位，如窗帘孔眼区域、电器控制面板），右侧约 50% 为卖点说明区：一个大号图标 + 一行短标题（≥36px）+ 2~3 行说明文字。图标与文字左对齐，信息层级清晰。背景使用浅色色块区分左右。 |
| **Mobile** | 纵向上下分区。上方约 45% 为产品局部特写，下方约 55% 为卖点说明：大号图标 + 短标题（≥48px）+ 1~2 行精简说明。文字比桌面版更大、更少，避免小屏文字拥挤。 |

#### 图 3：核心卖点 2

| 版式 | 详细画面内容与构图说明 |
|------|----------------------|
| **Desktop** | 结构与图 2 一致（左侧产品特写 + 右侧图标文字说明），但展示第二个卖点，特写区域聚焦不同部位。背景色块可换为次品牌色以区分图 2。保持与图 2 的版式节奏一致，形成系列感。 |
| **Mobile** | 结构与图 2 一致（上方特写 + 下方说明），文字进一步精简，每行说明不超过 3 个短词。保证小屏快速浏览即可抓住卖点。 |

#### 图 4：规格对比 / 信息图

| 版式 | 详细画面内容与构图说明 |
|------|----------------------|
| **Desktop** | 横向多列布局。可做横向对比表（3~4 列规格参数，每列一个维度如尺寸/材质/数量/配件），或规格信息图卡（多个卡片横向排列，每卡一个规格 + 大号数字标注）。数字标注用大号粗体（≥36px），纯数字优先。背景使用网格或卡片分隔，信息密度可比其他图更高。 |
| **Mobile** | 纵向信息列表。每行一个规格，左侧图标 + 中间规格名 + 右侧大号数值（≥48px）。行与行之间留足间距，每行信息精简。禁止横向多列（小屏无法横排），必须纵向单列排列。 |

#### 图 5：使用场景 / 套装内容

| 版式 | 详细画面内容与构图说明 |
|------|----------------------|
| **Desktop** | 宽幅场景图或套装平铺。若为场景图：产品置于真实使用环境（如窗帘挂满整面窗、电器在厨房台面运行），左右展示环境全景。若为套装平铺：所有配件在浅色背景上平铺排列，左右展开，标注"PACK OF N"。强调横向空间感与完整性。 |
| **Mobile** | 场景图或套装平铺，纵向居中展示。产品/套装在画面中央纵向排列，背景简洁。套装平铺时改为纵向堆叠或居中圆形排列，确保每个配件在小屏可见。套装数量标注用大号数字（≥48px）。 |

## 3. A+ 与 Listing 的关系

### 3.1 生成顺序

A+ 图在 Listing 图之后生成。完整顺序为：

```
Phase 0：锁定产品事实 + 生成保真锁定卡
  ↓
Phase 1：规划（Listing 9 张 + A+ Desktop 5 张 + A+ Mobile 5 张）
  ↓
Phase 2：生成 Listing 图 → 质检 Listing 图
  ↓
Phase 2（续）：生成 A+ Desktop 图 → 质检
  ↓
Phase 2（续）：生成 A+ Mobile 图 → 质检
  ↓
输出全部资产 + QA 报告
```

### 3.2 事实复用原则

A+ 图直接复用 Listing 阶段已锁定的内容，不得重新编造：

| 复用项 | 来源 | A+ 中的处理 |
|--------|------|------------|
| 产品不可变层 | `prompt_template.md` 第 2 节统一产品描述 | 原样引用，不改一字 |
| 产品锁段落 | `product_fidelity_lock.md` 第 5 节 | 原样包含，每张 A+ 图必须包含 |
| 保真锁定卡 | `product_fidelity_lock.md` 第 4 节 | 作为 A+ 质检基准 |
| 精度模式 | `product_fidelity_lock.md` 第 6 节 | A+ 图与 Listing 图保持同一模式 |
| 敏感词规则 | `claims_compliance.md` 115 词 | A+ 图同样适用，无例外 |

> 核心规则：A+ 图的产品不可变层和产品锁段落与 Listing 图完全一致。如果发现 A+ 视觉表达与产品事实冲突，应修改 A+ 视觉表达层，而非修改产品不可变层或产品锁。

### 3.3 用户可选范围

用户可选择只生成部分资产，灵活组合：

| 用户选择 | 生成内容 |
|----------|----------|
| 只生成 Listing 图 | 1 主图 + 副图（最多 9 张） |
| 只生成 A+ 图 | Desktop 5 张 + Mobile 5 张 |
| 两者都生成 | Listing 全套 + A+ Desktop + A+ Mobile |

> 注意：即使用户"只生成 A+ 图"，仍必须先完成 Phase 0（锁定产品事实 + 保真锁定卡）。A+ 图同样依赖已锁定的产品事实，不得跳过事实锁定直接生图。

### 3.4 精度模式一致性

A+ 图的精度模式（AI生图模式 / 混合模式）必须与 Listing 图保持一致。切换模式需在 Phase 1 规划阶段由用户明确决定，Phase 2 执行中不得中途切换。

- 若 Listing 用 AI生图模式 → A+ 也用 AI生图模式
- 若 Listing 用混合模式 → A+ 也用混合模式（混合模式指引同样适用 A+ 图）

## 4. Desktop 专属提示词模板

> Desktop 版的核心特征：横向构图、左右分区、宽幅场景、信息密度较高。
> 本节提供 Desktop 版的视觉表达层模板与完整示例。
> 产品不可变层、产品锁段落、合规约束层均复用 `prompt_template.md` 通用模板，此处不重复。

### 4.1 Desktop 视觉表达层模板

```
【视觉表达层 — Desktop A+ — 开始】
- 画幅：1464 × 600 px，横版 ~2.44:1
- 构图：横向左右分区
  · 左侧 {百分比}%：{左侧内容描述，如产品主体 / 局部特写}
  · 右侧 {百分比}%：{右侧内容描述，如场景 / 图标文字说明 / 规格卡片}
  · 左右之间用 {过渡方式，如品牌色块渐变 / 留白分隔} 自然衔接
- 信息层级：{如，产品为主体，文字标注在右侧留白区，自上而下排列}
- 文字标注：
  · 标注1：大号 {颜色} 粗体 "{短词}"，位于 {位置}（字号 ≥ 36px）
  · 标注2：中号 {颜色} 粗体 "{短词}"，位于 {位置}（字号 ≥ 36px）
  · 标注3：大号 {颜色} 粗体 "{数字}"，位于 {位置}（纯数字优先）
- 背景：{如，左侧品牌色块 #XXXXXX + 右侧浅灰渐变 / 生活场景}
- 光线：{如，柔光箱左上打光，强调横向延展感}
- 横向叙事：视觉重心从左到右流动，体现宽幅场景的延展性
【视觉表达层 — Desktop A+ — 结束】
```

### 4.2 Desktop 文字标注规范

- 英文字符字号至少 36px，推荐 48~72px
- 优先使用简单数字和短词（≤3 词）
- 文字横向排列，与左右分区结构对齐
- 避免长句，AI 生成长句拼写错误率高

### 4.3 Desktop 完整示例（窗帘产品 · 图 4 规格信息图）

> 以下示例复用 `prompt_template.md` 中的窗帘统一产品描述与产品锁段落。

```

布料有自然垂坠褶皱，厚度中等，不透光（blackout）。
顶部结构为 grommet top（金属孔眼顶部），共 8 个银色金属孔眼，
孔眼材质为银色不锈钢，拉丝缎面光泽，孔眼内径为 1.6 英寸，沿顶部均匀分布。
产品尺寸为 52 英寸 × 108 英寸（宽 × 长，每片）。包装内含 2 件窗帘。
材质为 100% 涤纶三层编织遮光面料。
包含：2 × 窗帘面板。不包含：窗帘杆、安装配件。
【产品不可变层 — 结束】

【产品锁 — 不可更改，每张图必须包含】
- EXACTLY 8 grommets per panel, total 16 grommets (2 panels)
  (NOT 6, NOT 7, NOT 9, NOT 10, NOT 12 grommets)
- GROMMET TOP ONLY
  (NOT rod pocket, NOT tab top, NOT back tab, NOT clip rings)
- 108 inches long, reaches floor
  (NOT dragging, NOT too short, NOT 84 inches, NOT 96 inches)
- Silver stainless steel grommets, brushed satin finish
  (NOT mirror polished, NOT chrome, NOT gold, NOT bronze)
- Dark gray fabric
  (NOT blue tint, NOT purple tint, NOT black)
【产品锁 — 结束】

【视觉表达层 — Desktop A+ — 开始】
- 画幅：1464 × 600 px，横版 ~2.44:1
- 构图：横向 4 列规格卡片布局
  · 第1列：尺寸卡片，产品局部图 + 大号数值
  · 第2列：孔眼卡片，孔眼特写 + 大号数值
  · 第3列：材质卡片，面料纹理 + 短词
  · 第4列：包装卡片，2 片平铺 + 大号数值
  · 卡片间用浅灰分隔线，卡片背景为白色，整体背景浅灰渐变
- 信息层级：每张卡片以大号数字为视觉焦点，短词辅助
- 文字标注：
  · 标注1：大号深灰粗体 "108"", 位于第1列卡片中部（标注长度，字号 64px）
  · 标注2：大号深灰粗体 "1.6IN", 位于第2列卡片中部（标注孔眼内径，字号 64px）
  · 标注3：大号深灰粗体 "100%", 位于第3列卡片中部（标注涤纶含量，字号 64px）
  · 标注4：大号深灰粗体 "2", 位于第4列卡片中部（标注包装片数，字号 72px）
- 背景：浅灰渐变 #F5F5F5，卡片白色 #FFFFFF
- 光线：顶部柔光，均匀照亮四张卡片，无强烈阴影
- 横向叙事：四列等宽排列，视觉重心从左到右扫读
【视觉表达层 — Desktop A+ — 结束】

【合规约束层 — 开始】
- 禁用表达：best, #1, top, energy saving, eco-friendly, 100% blackout（无测试报告时）
  （参考 references/claims_compliance.md 完整 115 词清单）
- Negative Prompt：
  blurry, low quality, distorted, watermark, signature,
  misspelled text, garbled letters, reversed text, overlapping text, tiny text,
  NOT rod pocket, NOT tab top, NOT back tab,
  NOT gold grommet, NOT bronze grommet, NOT mirror polished,
  NOT blue tint, NOT purple tint,
  NOT curtain rod, NOT hardware, NOT screws,
  cartoon, anime, 3d render, illustration, painting, sketch
【合规约束层 — 结束】

==================================================
默认尺寸：1464 × 600 px（横版 Desktop）
==================================================
```

## 5. Mobile 专属提示词模板

> Mobile 版的核心特征：纵向构图、上下分区、大字号文字、信息精简。
> Mobile 版必须独立构图，禁止缩放 Desktop 版。
> 产品不可变层、产品锁段落、合规约束层均复用 `prompt_template.md` 通用模板，此处不重复。

### 5.1 Mobile 视觉表达层模板

```
【视觉表达层 — Mobile A+ — 开始】
- 画幅：1600 × 1200 px，竖版 4:3
- 构图：纵向上下分区（独立构图，非桌面版缩放）
  · 上方 {百分比}%：{上方内容描述，如产品居中大图 / 局部特写}
  · 下方 {百分比}%：{下方内容描述，如场景背景 / 文字说明 / 规格列表}
  · 上下之间用 {过渡方式，如留白 / 色块} 自然衔接
- 信息层级：{如，产品居中为主体，文字在下方留白区，纵向单列排列}
- 文字标注：
  · 标注1：大号 {颜色} 粗体 "{短词}"，位于 {位置}（字号 ≥ 48px）

```

### 5.2 Mobile 信息精简原则

Mobile 版在手机小屏浏览，必须遵循信息精简原则：

| 原则 | Desktop | Mobile |
|------|---------|--------|
| 每图文字量 | 可较多（左右可排多段） | 精简（每区不超过 2 行） |
| 每行词数 | ≤3 词 | ≤2 词，优先纯数字 |
| 字号下限 | ≥36px | ≥48px |
| 布局方向 | 横向多列 | 纵向单列 |
| 信息密度 | 较高 | 较低（小屏可读性优先） |

## 6. Desktop vs Mobile 构图差异对照

> 注意：本表与 references/prompt_template.md 第8.4节内容一致，修改时需同步更新。

> 以下对照表说明同一主题在桌面版和移动版上的构图差异。
> 核心原则：主题对应，构图独立。Mobile 不得是 Desktop 的缩放版本。

| 主题 | Desktop 构图 | Mobile 构图 | 关键差异 |
|------|-------------|-------------|----------|
| 图1 品牌横幅 | 左右分区：左侧产品 + 右侧场景，横向叙事 | 上下分区：上方产品居中 + 下方场景，纵向聚焦 | 横向流动 vs 纵向聚焦；Mobile 产品必须居中独立 |
| 图2 核心卖点1 | 左右并列：左产品特写 + 右图标文字 | 上下排列：上产品特写 + 下图标文字 | 左右并列 vs 上下排列；Mobile 文字更大更少 |
| 图3 核心卖点2 | 左右并列（同图2结构，换卖点） | 上下排列（同图2结构，文字更精简） | 同上；Mobile 每行词数更少 |
| 图4 规格对比 | 横向多列：3~4 列卡片横向排列 | 纵向单列：每行一个规格，纵向列表 | 横向多列 vs 纵向单列；Mobile 禁止横向多列 |
| 图5 使用场景 | 宽幅场景：左右展开全景或套装平铺 | 纵向居中：场景或套装纵向居中展示 | 宽幅全景 vs 纵向居中；Mobile 套装改纵向堆叠 |

### 6.1 通用差异原则

| 维度 | Desktop | Mobile |
|------|---------|--------|
| 分区方向 | 左右分区为主 | 上下分区为主 |
| 信息排列 | 横向多列可接受 | 纵向单列为唯一选择 |
| 产品位置 | 左侧或左侧+右侧 | 居中 |
| 文字字号 | ≥36px | ≥48px |
| 文字密度 | 较高 | 较低 |
| 场景展现 | 宽幅全景 | 聚焦居中 |
| 负空间 | 左右留白 | 上下留白 |

> 重要提醒：Mobile 版的纵向单列布局不是可选项，而是硬性要求。手机屏幕宽度有限，横向多列在小屏会导致文字过小不可读，违反 Mobile 字号 ≥48px 规范。

## 7. A+ 内容禁止事项

> A+ 内容的禁止事项分为两类：通用禁止（与 Listing 图一致的合规要求）和 A+ 专属禁止（A+ 版式特有要求）。

### 7.1 通用禁止事项（与 Listing 一致）

- [ ] **禁止使用 115 个敏感词**
  - A+ 图中所有文字同样适用 `references/claims_compliance.md` 的完整清单
  - 常见高风险：best, #1, top, FDA, eco-friendly, energy saving, guaranteed 等
  - 出现任一敏感词 = FAIL，无例外

- [ ] **禁止虚假声明**
  - 所有功能声明必须有证据支持
  - 无遮光测试报告不得标注 "100% BLACKOUT"（可改为 "ROOM DARKENING"）
  - 无认证不得出现 UL / ETL / CE / RoHS 标识

- [ ] **禁止展示竞品**
  - A+ 图中不得出现任何竞品品牌名称或 Logo
  - 对比图中不得指名道姓对比特定竞品（亚马逊 A+ 政策禁止直接对比竞品）

### 7.2 A+ 专属禁止事项

- [ ] **禁止移动版直接缩放桌面版**
  - Mobile 版必须独立构图，不得通过缩放/裁切 Desktop 版生成
  - 缩放会导致文字过小、左右分区在小屏失效
  - Mobile 版必须重新设计构图，独立生成

- [ ] **禁止跨版式复用构图比例**
  - Desktop 的左右分区比例（如左40%右60%）不得直接套用到 Mobile
  - Mobile 必须改为上下分区，重新分配空间比例

- [ ] **禁止 Mobile 版使用横向多列布局**
  - 手机屏宽有限，横向多列导致文字 <48px，违反可读性要求
  - 规格对比图在 Mobile 必须改为纵向单列

- [ ] **禁止 A+ 图产品事实与 Listing 图不一致**
  - A+ 图的产品不可变层、产品锁段落必须与 Listing 图完全一致
  - 不得因 A+ 场景不同而改变产品颜色、数量、结构、材质

- [ ] **禁止 A+ 图超出亚马逊 A+ 内容政策**
  - 不得在 A+ 中添加购买按钮、促销价格、折扣信息
  - 不得在 A+ 中重复 Listing 标题/五点描述的原文
  - 不得在 A+ 中放置外部链接或二维码
  - 不得在 A+ 中使用超过 2 个公司 Logo

- [ ] **禁止文字标注使用长句或复杂专有名词**
  - A+ 文字标注同样遵循 `prompt_template.md` 第 4 节规范
  - 每段不超过 3 个单词，避免 AI 拼写错误

## 8. A+ 质检要点摘要

> 以下为 A+ 图质检的关键检查项摘要。详细的逐项清单见 `references/qc_checklist.md` 第 8 节（A+ 专属质检清单）。
> A+ 图质检同样遵循 PASS / CONDITIONAL / FAIL 三级状态，判定逻辑与 Listing 图一致。

### 8.1 A+ 专属质检关键项

| 检查维度 | 关键检查项 | 不通过后果 |
|----------|-----------|-----------|
| 版式尺寸 | Desktop 1464×600 或 1940×1200 / Mobile 1600×1200 是否正确 | FAIL（不可修复）/ CONDITIONAL（可裁切） |
| 版式构图 | Mobile 是否独立构图（非 Desktop 缩放） | FAIL |
| 版式构图 | Mobile 是否纵向单列布局（非横向多列） | FAIL |
| 文字字号 | Desktop ≥36px / Mobile ≥48px | CONDITIONAL（可叠加文字层） |
| 产品事实 | 产品不可变层、产品锁段落是否与 Listing 一致 | FAIL |
| 可量化细节 | 孔眼/按钮等数量、结构、材质光泽是否与保真锁定卡一致 | FAIL |
| 合规 | 是否出现 115 敏感词 | FAIL |
| 合规 | 是否出现竞品、虚假声明、促销信息 | FAIL |

### 8.2 A+ 质检流程

A+ 图质检流程在 Listing 图质检流程基础上增加版式检查：

```
步骤1：读取生成的 A+ 图片，确认尺寸（Desktop 1464×600 或 1940×1200 / Mobile 1600×1200）
  ↓
步骤2：版式专项检查
  → Mobile 版是否独立构图（与 Desktop 对比，确认非缩放）
  → Mobile 版是否纵向单列布局
  → 文字字号是否达标（Desktop ≥36px / Mobile ≥48px）
  ↓
步骤3：产品事实检查（与 Listing 图一致，对照保真锁定卡）
  ↓
步骤4：合规检查（115 敏感词 + 竞品 + 虚假声明）
  ↓
步骤5：技术检查（拉伸变形、文字拼写、安全区裁切）
  ↓
步骤6：判定状态（取五维度最严重值）
```

### 8.3 A+ 质检与 Listing 质检的关系

| 维度 | Listing 图质检 | A+ 图质检 |
|------|---------------|----------|
| 产品事实检查 | 基准（保真锁定卡） | 复用同一基准，必须一致 |
| 合规检查 | 115 敏感词 | 相同 |
| 技术检查 | 尺寸 1600×1600 | 尺寸 Desktop 1464×600 或 1940×1200 / Mobile 1600×1200 |
| 版式检查 | 无 | A+ 专属：独立构图、纵向单列、字号下限 |
| 诚实降级 | 3 次返工未达标 → CONDITIONAL | 相同 |

> A+ 图同样适用诚实降级协议（见 `references/product_fidelity_lock.md` 第 7 节）。3 次返工仍无法通过保真质检时，必须标记 CONDITIONAL，如实告知偏差，不得假装通过。
