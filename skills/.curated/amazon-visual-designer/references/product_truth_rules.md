# 产品事实锁定规则

> 本文件定义"产品事实"的锁定规则，是 SKILL.md 中 Hard Rule 6（产品事实不可变）的详细说明。
> 一旦锁定产品事实（品牌、类型、颜色、尺寸、数量、结构、配件），所有后续步骤不得改变。
> 产品事实是整个视觉设计的"地基"，地基不稳则上层策划全部不可靠。

## 1. 不可变字段清单

> 以下字段一旦在 Phase 0/Phase 1 中锁定，所有后续步骤（策划、Prompt 生成、图片生成、质检）
> 都必须严格遵循，不得擅自修改。
> 如需修改，必须回到 Phase 0 重新验证并提供新证据。

### 1.1 字段总表

| 字段 | 说明 | 错误示例 | 锁定后影响范围 |
|------|------|----------|------------|
| **品牌** | 产品的品牌名称。未提供时标注"未提供"，不得自动添加。 | 用户提供的商品信息未提品牌，但 Prompt 中自动加了"BrandX"的 Logo | 所有图片中的品牌呈现、Logo 使用、品牌色应用 |
| **产品类型** | 产品的品类归属，如"厨房收纳架""蓝牙耳机"等。 | 实际是"收纳架"，但策划中写成"置物架"导致类目偏差 | 视觉叙事方向、场景选择、竞品对标 |
| **颜色/图案** | 产品的实际颜色和表面图案，包括哑光/亮面等质感描述。 | 锁定为"哑光黑"，但生成图中产品变成了"亮面黑" | 所有图片中产品的颜色表现 |
| **尺寸/比例** | 产品的物理尺寸（长×宽×高）或关键比例关系。 | 用户提供的商品信息写 54×18cm，Prompt 中按 40×15cm 的比例生成 | 尺寸图、对比图、场景图中的产品比例 |
| **包装数量** | 单个包装内包含的产品件数，如"1件""6件套"等。 | 锁定为"6件套"，但生成图中只出现了4件 | 所有展示包装内容的图片 |
| **结构** | 产品的结构特征，如"可折叠""可拆卸""一体成型"等。 | 锁定为"可折叠"，但生成图显示为"固定结构" | 功能图、结构展示图 |
| **包含/不包含** | 包装内明确包含和不包含的物品清单。 | 锁定"不含螺丝"，但生成图中出现了螺丝配件 | 配件展示图、包装内容图 |
| **材质** | 产品的主要材质，如"304不锈钢""ABS塑料""碳钢"等。 | 锁定为"碳钢"，但 Prompt 中写成"不锈钢"暗示更高级别 | 材质展示图、文字标注中的材质描述 |

### 1.2 各字段详细说明

#### 1.2.1 品牌

- **定义**：产品的品牌名称，包括品牌 Logo 和品牌色。
- **锁定来源**：用户提供的商品信息中的品牌字段、用户提供的品牌素材。
- **锁定规则**：
  - 如有品牌，必须从用户提供的品牌素材获取 Logo 和品牌色，不得自行设计。
  - 如未提供品牌，锁定为"未提供"，所有图片不得出现任何品牌 Logo。
  - 品牌名称的拼写必须与原始资料完全一致。
- **错误示例**：
  - 用户提供的商品信息未提品牌，但 Prompt 中自动加了"BrandX"的 Logo -> 违规
  - 用户提供的品牌素材中 Logo 是红色，但生成图中用了蓝色 Logo -> 违规

#### 1.2.2 产品类型

- **定义**：产品的品类归属，决定了视觉叙事的方向和场景选择。
- **锁定来源**：用户提供的商品信息中的产品类型字段。
- **锁定规则**：
  - 产品类型必须与实际产品一致，不得为了"好看"而调整品类描述。
  - 如产品类型有歧义（如既是"收纳架"也是"书架"），以用户确认为准。
- **错误示例**：
  - 实际是"厨房收纳架"，但策划中写成"通用置物架"导致场景选择错误 → 违规

#### 1.2.3 颜色/图案

- **定义**：产品的实际颜色和表面图案，包括哑光/亮面等质感描述。
- **锁定来源**：用户提供的商品信息中的颜色字段、产品底图中的真实产品图。
- **锁定规则**：
  - 颜色必须以真实产品图为准，文字描述为辅。
  - 如文字写"哑光黑"但图片显示"亮面黑"，触发冲突检测。
  - 图案（如有花纹、纹理）必须如实描述，不得添加或去除。
- **错误示例**：
  - 锁定为"哑光黑"，但生成图中产品变成了"亮面黑" → 违规
  - 产品无图案，但生成图中添加了花纹 → 违规

#### 1.2.4 尺寸/比例

- **定义**：产品的物理尺寸（长×宽×高）或关键比例关系。
- **锁定来源**：用户提供的商品信息中的尺寸字段、包装图片（如有）中的包装标注、补充资料（如有）中的规格书。
- **锁定规则**：
  - 尺寸必须以最高证据等级的来源为准（规格书 > 包装标注 > 用户提供的商品信息）。
  - 如不同来源的尺寸不一致，触发冲突检测。
  - 尺寸图中的标注必须与锁定值完全一致。
  - 场景图中产品的比例关系必须符合实际尺寸比例。
- **错误示例**：
  - 用户提供的商品信息写 54×18cm，Prompt 中按 40×15cm 的比例生成 -> 违规
  - 尺寸图标注重径"20cm"，但实际锁定值为"18cm" → 违规

#### 1.2.5 包装数量

- **定义**：单个包装内包含的产品件数，如"1件""6件套"等。
- **锁定来源**：用户提供的商品信息中的包装数量字段、包装图片（如有）中的包装标注。
- **锁定规则**：
  - 包装数量必须以实际包装图或规格书为准。
  - 如文字写"6件套"但包装图标"4件套"，触发冲突检测。
  - 所有展示包装内容的图片中，产品件数必须与锁定值一致。
- **错误示例**：
  - 锁定为"6件套"，但生成图中只出现了4件 → 违规
  - 锁定为"1件"，但生成图中出现了多个产品暗示多件装 → 违规

#### 1.2.6 结构

- **定义**：产品的结构特征，如"可折叠""可拆卸""一体成型"等。
- **锁定来源**：用户提供的商品信息中的结构字段、产品底图中的真实产品图。
- **锁定规则**：
  - 结构特征必须以真实产品图为准，文字描述为辅。
  - 如文字写"可折叠"但图片显示"固定结构"，触发冲突检测。
  - 功能图和结构展示图必须如实反映产品的结构特征。
  - 不得在图片中展示产品不具备的结构功能。
- **错误示例**：
  - 锁定为"可折叠"，但生成图显示为"固定结构" → 违规
  - 产品不可拆卸，但生成图中展示了"拆卸"过程 → 违规

#### 1.2.7 包含/不包含

- **定义**：包装内明确包含和不包含的物品清单。
- **锁定来源**：用户提供的商品信息中的包含/不包含字段、包装图片（如有）中的包装内容图。
- **锁定规则**：
  - 包含清单必须如实列出所有包装内物品。
  - 不包含清单用于明确"不含什么"，避免消费者误解。
  - 配件展示图中的物品必须与包含清单一致。
  - 不得在图片中出现不包含清单中的物品（即使是作为"对比"也不行）。
- **错误示例**：
  - 锁定"不含螺丝"，但生成图中出现了螺丝配件 → 违规
  - 锁定"含安装说明书"，但生成图中没有展示说明书 → 需补充

#### 1.2.8 材质

- **定义**：产品的主要材质，如"304不锈钢""ABS塑料""碳钢"等。
- **锁定来源**：用户提供的商品信息中的材质字段、补充资料（如有）中的规格书或测试报告。
- **锁定规则**：
  - 材质必须以最高证据等级的来源为准（测试报告 > 规格书 > 用户提供的商品信息）。
  - 如不同来源的材质描述不一致（如"304不锈钢"vs"201不锈钢"），触发冲突检测。
  - 材质描述不得"升级"（如碳钢不得写成不锈钢）。
  - 文字标注中的材质描述必须与锁定值完全一致。
- **错误示例**：
  - 锁定为"碳钢"，但 Prompt 中写成"不锈钢"暗示更高级别 → 违规
  - 锁定为"ABS塑料"，但生成图中产品呈现金属质感 → 违规

## 2. 证据等级体系

> 产品事实的每一个字段都需要标注证据等级。
> 证据等级决定了该字段的"可信度"，也决定了该字段能否用于公开表达。

### 2.1 证据等级定义

| 等级 | 名称 | 定义 | 来源示例 | 可否公开表达 |
|------|------|------|----------|------------|
| **A** | 强证据 | 经第三方或权威机构验证的文件 | 测试报告、认证证书、官方规格书 | 可以公开表达，可用于图片文字标注 |
| **B** | 产品事实 | 产品本身的实物证据 | 材质标签、实物尺寸测量、包装内容、真实结构照片 | 可以公开表达，可用于图片文字标注 |
| **C** | 卖家说明 | 卖家或供应商提供的说明，未经独立验证 | 供应商口头说明、未经核验的产品文案、电商页面描述 | 谨慎使用，不建议直接用于图片文字标注，需标注"卖家说明" |
| **D** | 模型推测 | AI 模型从图片或描述中推测的内容 | 从图片猜材质、从外观猜性能 | 不得公开表达，仅用于内部参考 |

### 2.2 各等级详细说明

#### 2.2.1 A 级：强证据

- **定义**：经第三方或权威机构验证的文件，具有最高可信度。
- **来源示例**：
  - 第三方实验室测试报告（如 SGS、TUV 报告）
  - 认证证书（如 FDA、CE、FCC 认证）
  - 官方规格书（由制造商出具并盖章）
  - 专利文件
- **使用规则**：
  - 可以直接用于图片文字标注（如"FDA 认证""304 不锈钢"）。
  - 可以用于性能/安全声明。
  - 在 `approved_image_plan.json` 中的 `product_truth` 中标注为"A 级"。
- **注意事项**：
  - 即使是 A 级证据，也需要确认证书/报告与当前产品型号匹配。
  - 证书的有效期需在检查范围内。

#### 2.2.2 B 级：产品事实

- **定义**：产品本身的实物证据，通过观察和测量获得。
- **来源示例**：
  - 材质标签（产品上的材质标注）
  - 实物尺寸测量（用尺子量的尺寸）
  - 包装内容实物（打开包装看到的实际内容）
  - 真实结构照片（白底图显示的结构特征）
- **使用规则**：
  - 可以用于图片文字标注（如尺寸、数量、结构描述）。
  - 可以用于产品事实展示（如"可折叠""6 件套"）。
  - 在 `approved_image_plan.json` 中的 `product_truth` 中标注为"B 级"。
- **注意事项**：
  - B 级证据的可靠性取决于测量的准确性，建议交叉验证。
  - 如材质仅有标签无测试报告，标注为 B 级而非 A 级。

#### 2.2.3 C 级：卖家说明

- **定义**：卖家或供应商提供的说明，未经独立验证。
- **来源示例**：
  - 供应商口头说明（如"我们用的是 304 不锈钢"）
  - 未经核验的产品文案（如电商页面的产品描述）
  - 卖家自行填写的商品信息（无实物佐证的部分）
- **使用规则**：
  - 谨慎使用，不建议直接用于图片文字标注。
  - 如需使用，必须标注"卖家说明"或"以实物为准"。
  - 在 `approved_image_plan.json` 中的 `product_truth` 中标注为"C 级"，并加告警。
  - 建议向用户确认是否可以升级为 B 级（提供实物证据）或 A 级（提供测试报告）。
- **注意事项**：
  - C 级证据的字段在"待确认问题列表"中列为"未确认字段"。
  - 不得将 C 级证据的字段用于性能/安全声明。

#### 2.2.4 D 级：模型推测

- **定义**：AI 模型从图片或描述中推测的内容，无任何来源佐证。
- **来源示例**：
  - 从图片猜材质（如"看起来像不锈钢"）
  - 从外观猜性能（如"这个结构应该很稳固"）
  - 从颜色猜工艺（如"这个颜色可能是电镀"）
- **使用规则**：
  - **不得公开表达**，即不得出现在任何图片文字标注、Prompt 的视觉表达层、或交付给用户的文案中。
  - 仅用于内部参考和分析。
  - 在 `approved_image_plan.json` 中的 `product_truth` 中标注为"D 级"，并加告警。
  - D 级推测必须列出推测依据，供后续验证。
- **注意事项**：
  - D 级推测不得作为锁定值，必须标注为"未知"并要求用户提供证据。
  - 如用户确认 D 级推测正确，需提供 B 级或以上证据才能升级。

### 2.3 证据等级升降规则

```
D 级（模型推测）
  ↓ 用户提供 B 级证据（实物照片/标签）
B 级（产品事实）
  ↓ 用户提供 A 级证据（测试报告/证书）
A 级（强证据）

逆向降级：
  - 如发现 A 级证据与实物不符 → 降级为 D 级，触发冲突检测
  - 如发现 B 级证据与卖家说明矛盾 → 以 B 级为准，C 级标记为"矛盾"
```

### 2.4 证据等级使用决策表

| 使用场景 | 最低要求证据等级 | 说明 |
|----------|----------------|------|
| 图片文字标注中的材质描述 | B 级 | 必须有实物标签或规格书佐证 |
| 图片文字标注中的尺寸标注 | B 级 | 必须有实物测量或包装标注 |
| 图片文字标注中的认证声明（如 FDA） | A 级 | 必须有证书，不得仅有卖家说明 |
| 图片文字标注中的性能声明（如"承重 50kg"） | A 级 | 必须有测试报告 |
| 内部分析和推测 | D 级可用 | 但不得出现在任何对外输出中 |
| 竞品分析 | 不适用 | 竞品信息仅用于研究，不用于本产品标注 |

## 3. 品牌隔离规则

> 品牌隔离规则确保不同品牌的素材不混用，避免侵权和误导。

### 3.1 未提供品牌时不自动添加

- **规则**：当用户提供的商品信息和用户提供的品牌素材均未提供品牌信息时，所有生成的图片中不得出现任何品牌 Logo、品牌名称或品牌标识。
- **原因**：自动添加品牌可能导致以下问题：
  - 添加了错误品牌 → 侵权风险
  - 添加了虚构品牌 → 违反"不可伪造"原则
  - 添加了无关品牌 → 误导消费者
- **处理方式**：
  - 在 `approved_image_plan.json` 中的 `product_truth` 中品牌字段标注"未提供"。
  - 在视觉策略中品牌一致性部分标注"未提供品牌，使用中性视觉风格"。
  - 所有 Prompt 中不得包含品牌 Logo 相关描述。

### 3.2 不同品牌不得混用 Logo 和颜色

- **规则**：当产品有明确品牌时，只能使用该品牌的 Logo 和品牌色，不得混入其他品牌的元素。
- **原因**：
  - 混用 Logo → 侵权风险
  - 混用品牌色 → 品牌识别混乱
  - 混用品牌素材 → 违反品牌一致性
- **具体要求**：
  - Logo 来源：只能从用户提供的品牌素材获取，不得从网络下载或自行设计。
  - 品牌色来源：只能从用户提供的品牌素材中获取，不得自行选择。
  - 竞品 Logo：不得出现在任何生成的图片中。
  - 竞品品牌色：不得用于本产品的视觉设计。
- **错误示例**：
  - 产品品牌是 Brand A，但生成图中出现了 Brand B 的 Logo → 违规
  - 产品品牌是 Brand A（红色系），但生成图用了竞品 Brand B 的蓝色系 → 违规
  - 对比图中出现了竞品的 Logo → 违规

### 3.3 竞品图片只能用于研究，不得作为本产品源图

- **规则**：竞品参考图片只能用于市场视觉研究，不得作为本产品图片生成的参考源图或直接素材。
- **原因**：
  - 直接使用竞品图片 → 侵权风险
  - 将竞品图片作为源图 → 可能导致生成图中出现竞品品牌元素
  - 将竞品图片作为参考 → 可能导致产品事实偏差（竞品与本产品不同）
- **具体要求**：
  - 竞品图片只能在市场视觉研究中被引用和分析。
  - 竞品图片不得出现在 `approved_image_plan.json` 的 `reference_image` 字段中。
  - 竞品图片不得作为图片生成 API 的输入。
  - 竞品的视觉风格可以借鉴（如"白底实景"风格），但竞品的具体图片不得使用。
- **错误示例**：
  - 将竞品主图直接作为本产品主图的参考源图 → 违规
  - 在生成 Prompt 中引用竞品图片路径 → 违规
  - 将竞品图片中的产品直接"复制"到本产品图片中 → 违规

### 3.4 品牌素材使用检查清单

在 Phase 1 策划阶段，必须对品牌素材的使用进行以下检查：

| 检查项 | 通过条件 | 不通过处理 |
|--------|----------|------------|
| 品牌一致性 | 所有图片中的品牌 Logo 和品牌色来源一致 | 告警，统一来源 |
| Logo 准确性 | Logo 与用户提供的品牌素材中的完全一致 | 告警，更换 Logo |
| 品牌色准确性 | 品牌色与用户提供的品牌素材中的完全一致 | 告警，更换品牌色 |
| 竞品隔离 | 生成图片中无任何竞品品牌元素 | 告警，移除竞品元素 |
| 无品牌处理 | 未提供品牌时，图片中无任何品牌标识 | 告警，移除品牌标识 |

## 4. 冲突处理规则

> 资料冲突是指同一字段在不同来源中出现了不一致的描述。
> 冲突不解决，不进入 Phase 1 后续步骤。

### 4.1 冲突类型与示例

| 冲突类型 | 示例 | 严重程度 | 处理方式 |
|----------|------|----------|----------|
| **尺寸冲突** | 用户提供的商品信息写 54×18cm，包装图标注 52×16cm | 高 | 停止，要求用户确认 |
| **颜色冲突** | 用户提供的商品信息写"哑光黑"，白底图显示"亮面黑" | 高 | 停止，要求用户确认 |
| **数量冲突** | 用户提供的商品信息写"6 件套"，包装图标注"4 件套" | 高 | 停止，要求用户确认 |
| **材质冲突** | 用户提供的商品信息写"304 不锈钢"，供应商口头说"201 不锈钢" | 高 | 停止，要求用户确认 |
| **品牌冲突** | 用户提供的商品信息写 Brand A，品牌素材里是 Brand B | 高 | 停止，要求用户确认 |
| **结构冲突** | 用户提供的商品信息写"可折叠"，白底图显示"固定结构" | 高 | 停止，要求用户确认 |
| **配件冲突** | 用户提供的商品信息写"含螺丝"，包装内容图无螺丝 | 中 | 停止，要求用户确认 |
| **类型冲突** | 用户提供的商品信息写"收纳架"，规格书写"置物架" | 低 | 以更具体的描述为准，告警 |

### 4.2 冲突处理流程

```
步骤1：发现冲突
  - 在 Phase 0 输入验证阶段，交叉比对所有来源
  - 发现同一字段在不同来源中数值/描述不一致
  ↓
步骤2：记录冲突
  - 在 `approved_image_plan.json` 中的 `product_truth` 的"冲突记录"表中记录
  - 在"待确认问题列表"的"冲突项"表中记录
  ↓
步骤3：停止流程
  - 冲突字段达到 1 个或以上 → 停止进入 Phase 1
  - 向用户展示冲突详情
  ↓
步骤4：等待用户裁决
  - 用户指定以哪个来源为准
  - 或用户提供新证据解决冲突
  ↓
步骤5：更新锁定值
  - 根据用户裁决，更新 `approved_image_plan.json` 中的 `product_truth` 中的锁定值
  - 标注证据等级和最终来源
  - 冲突状态更新为"已解决"
  ↓
步骤6：继续流程
  - 所有冲突解决后，继续进入 Phase 1
```

### 4.3 冲突处理原则

#### 4.3.1 证据等级优先原则

当不同来源的描述冲突时，以证据等级更高的来源为准：

```
A 级（强证据）> B 级（产品事实）> C 级（卖家说明）> D 级（模型推测）
```

- 如测试报告（A 级）与卖家说明（C 级）冲突 → 以测试报告为准。
- 如实物标签（B 级）与供应商口头说明（C 级）冲突 → 以实物标签为准。
- 如两个同等级来源冲突 → 必须由用户裁决，不得自行选择。

#### 4.3.2 明确优先于模糊原则

当同一字段一个来源明确、另一个来源模糊时，以明确来源为准：

- 用户提供的商品信息写"54×18×25cm"（明确），包装图标注"约 50cm"（模糊）-> 以用户提供的商品信息为准，但需告警。
- 用户提供的商品信息写"不锈钢"（模糊，未指明型号），规格书写"304 不锈钢"（明确）-> 以规格书为准。

#### 4.3.3 实物优先于文字原则

当文字描述与实物图片冲突时，以实物图片为准：

- 用户提供的商品信息写"哑光黑"（文字），白底图显示"亮面黑"（实物）-> 以白底图为准，但需告警并要求用户确认。
- 用户提供的商品信息写"6 件套"（文字），包装内容图显示"4 件"（实物）-> 以包装内容图为准，但需告警并要求用户确认。

> 注意：实物优先于文字的前提是实物图片清晰可靠。如实物图片模糊或角度不清，不得作为优先依据。

#### 4.3.4 不确定时不锁定原则

当冲突无法通过上述原则解决时，该字段标注为"未确认"，不锁定：

- 两个同等级来源冲突且无法判断 → 标注"未确认"。
- 实物图片模糊且文字描述矛盾 → 标注"未确认"。
- 用户未回复确认 → 标注"未确认"。

> "未确认"字段不得用于图片文字标注，必须在"待确认问题列表"中列为阻断项。

### 4.4 需要用户确认的场景

以下场景必须要求用户确认，不得自行决定：

| 场景 | 需要确认的内容 | 确认方式 |
|------|------------|----------|
| 同等级来源冲突 | 选择哪个来源为准 | 用户指定来源 |
| 文字与实物冲突 | 确认以哪个为准 | 用户确认或提供新证据 |
| 字段完全缺失 | 确认该字段的值 | 用户提供信息 |
| 证据等级为 C/D 级 | 确认是否可以用于公开表达 | 用户确认或提供更高等级证据 |
| 用户修改意见涉及产品事实变更 | 确认变更并提供新证据 | 用户提供新证据 |
| 品牌信息缺失但需使用品牌 | 确认品牌信息 | 用户提供品牌素材 |

### 4.5 冲突记录模板

在产品特征摘要表中，冲突记录表格式如下：

```markdown
## 冲突记录

| 序号 | 字段 | 来源1（等级） | 来源2（等级） | 冲突描述 | 处理方式 | 状态 |
|------|------|------------|------------|----------|----------|------|
| 1 | 尺寸 | 商品信息: 54×18cm（C级） | 包装图: 52×16cm（B级） | 长宽均不一致 | 以包装图为准（B级优先） | 已解决 |
| 2 | 材质 | 商品信息: 304不锈钢（C级） | 规格书: 201不锈钢（A级） | 材质型号不一致 | 以规格书为准（A级优先） | 已解决 |
| 3 | 颜色 | 商品信息: 哑光黑（C级） | 白底图: 亮面黑（B级） | 质感描述不一致 | 待用户确认 | 未解决 |
```

### 4.6 冲突处理检查清单

在进入 Phase 1 之前，必须确认以下检查项全部通过：

- [ ] 所有字段的来源已确认
- [ ] 所有字段的证据等级已标注
- [ ] 所有冲突已记录
- [ ] 所有冲突已解决（或标注为"未确认"并列为阻断项）
- [ ] "未确认"字段已在"待确认问题列表"中列出
- [ ] 用户已对关键冲突做出裁决（如有）

> 以上检查项未全部通过前，不得进入 Phase 1 策划阶段。
