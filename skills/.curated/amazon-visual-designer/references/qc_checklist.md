# 质检标准清单（QC Checklist）

> 本文件是 Amazon Visual Designer 的核心参考文件之一。
> 当需要执行图片质检（Quality Check）时加载本文件。
>
> 设计原则：
> 1. 高内聚低耦合 -- 产品事实质检、保真细节质检、合规质检、技术质检、文字标注质检五个维度独立检查，互不依赖。
> 2. 不可伪造 -- 质检以已锁定的产品事实（`approved_image_plan.json` 中的 `product_truth`）为基准，
>    不得因图片"好看"而放宽事实要求。
> 3. 逐张闭环 —— 每张图生成后必须完成质检，不合格则修复后重新质检，不得跳过。

## 1. QA 状态定义

每张图质检完成后，必须给出以下三种状态之一。状态判定基于五个维度的综合结果。

### 状态一览表

| 状态 | 含义 | 判定条件 | 后续处理 |
|------|------|----------|----------|
| **PASS** | 产品事实、保真细节、合规、技术和文字标注检查全部通过 | 五个维度清单全部通过 | 可交付，纳入最终输出 |
| **CONDITIONAL** | 存在可修复问题，不影响核心事实 | 产品事实核心项通过，但存在文字拼写、排版、轻微偏色等可修复问题 | 修复后重新质检，直到 PASS |
| **FAIL** | 存在关键错误 | 品牌、数量、结构、合规等关键项不通过 | 禁止交付，必须修改 Prompt 重新生成 |

### 状态判定决策树

```
质检开始
  │
  ├─ 产品事实质检清单（第3节）
  │    ├─ 品牌名称错误？ → FAIL
  │    ├─ 产品类型错误？ → FAIL
  │    ├─ 颜色/图案与实物不一致？ → FAIL
  │    ├─ 尺寸/比例严重错误？ → FAIL
  │    ├─ 包装数量错误？ → FAIL
  │    ├─ 结构错误（如孔眼类型、控制面板布局）？ → FAIL
  │    ├─ 配件错误（包含不该有的，或缺少该有的）？ -> FAIL
  │    └─ 全部通过 -> 继续
  │
  ├─ 保真细节质检清单（第3.5节）
  │    ├─ 可数部件数量与保真锁定卡不一致？ -> FAIL
  │    ├─ 结构方式与保真锁定卡不一致？ -> FAIL
  │    ├─ 尺寸比例与保真锁定卡不一致？ -> FAIL
  │    ├─ 材质光泽与保真锁定卡不一致？ -> FAIL
  │    └─ 全部通过 -> 继续
  │
  ├─ 合规质检清单（第4节）
  │    ├─ 出现 115 个敏感词中的任何一个？ -> FAIL
  │    ├─ 出现绝对化用语（best, #1, top 等）？ → FAIL
  │    ├─ 出现未经证实的功能声明？ → FAIL
  │    ├─ 出现竞品品牌名称？ → FAIL
  │    └─ 全部通过 → 继续
  │
  ├─ 技术质检清单（第5节）
  │    ├─ 图片尺寸错误且无法裁切修复？ → FAIL
  │    ├─ 严重拉伸变形？ → FAIL
  │    ├─ 文字不可读且无法修复？ → CONDITIONAL（可叠加文字层修复）
  │    ├─ 安全区内有关键内容被裁切？ → CONDITIONAL（可重新裁切）
  │    ├─ 轻微偏色？ → CONDITIONAL（可调色修复）
  │    ├─ 文字拼写错误？ -> CONDITIONAL（可叠加文字层修复）
  │    └─ 全部通过 -> 继续
  │
  ├─ 文字标注质检
  │    ├─ 文字拼写错误且无法修复？ -> CONDITIONAL（可叠加文字层修复）
  │    ├─ 文字字号不足 36px？ -> CONDITIONAL（可叠加文字层修复）
  │    ├─ 文字位置在安全区外或被产品遮挡？ -> CONDITIONAL（可调整构图修复）
  │    └─ 全部通过 -> PASS
  │
  └─ 最终状态：取五维度中最严重的状态
```

> 注意：五个维度的状态取"最严重"值。即只要有一个维度 FAIL，整张图即 FAIL；
> 如果没有 FAIL 但有 CONDITIONAL，整张图为 CONDITIONAL；
> 全部 PASS 才是 PASS。

## 2. 逐张质检流程

每张图生成后，必须按以下步骤完成质检。不得跳过任何步骤。

### 质检步骤

```
步骤 1：读取生成的图片
  → 加载刚生成的图片文件
  → 记录文件名、尺寸、格式
  → 确认图片可正常打开，无损坏

步骤 2：与参考图/主图对比
  → 并排展示生成图与真实产品白底参考图
  → 从整体到细节逐一对比
  → 重点关注：形状、颜色、部件数量、结构特征

步骤 3：检查产品事实一致性
  -> 打开 approved_image_plan.json 中的 product_truth（已锁定的产品事实）
  -> 逐项核对第 3 节"产品事实质检清单"
  → 任何不一致项记录为问题

步骤 4：检查敏感词
  -> 检查图中所有可见文字
  -> 对照 references/claims_compliance.md 中的 115 个敏感词清单
  -> 检查绝对化用语、功能声明、竞品品牌
  -> 逐项核对第 4 节"合规质检清单"

步骤 5：检查文字标注
  → 检查图中所有文字的拼写是否正确
  → 检查字号是否 ≥ 36px
  → 检查文字位置是否在安全区内
  → 检查文字是否被产品遮挡

步骤 6：判定状态
  -> 汇总五个维度的检查结果
  -> 按第 1 节决策树判定 PASS / CONDITIONAL / FAIL
  → 记录判定结果和问题清单
  → 如为 CONDITIONAL → 修复后回到步骤 1 重新质检
  → 如为 FAIL → 修改 Prompt 重新生成，再回到步骤 1
  → 如为 PASS → 标记为可交付，进入下一张图
```

### 质检记录模板

每张图的质检结果应记录以下信息：

```
【质检记录】
- 图片编号：{如 listing-03}
- 文件名：{如 listing-03.jpg}
- 尺寸：{如 1600 × 1600}
- 质检时间：{时间戳}

- 产品事实检查：{PASS / FAIL} — {问题描述}
- 合规检查：{PASS / FAIL} — {问题描述}
- 技术检查：{PASS / CONDITIONAL / FAIL} — {问题描述}

- 最终状态：{PASS / CONDITIONAL / FAIL}
- 修复措施：{如需要}
- 重新质检次数：{数字}
```

## 3. 产品事实质检清单

> 以下清单基于已锁定的产品事实（`approved_image_plan.json` 中的 `product_truth`）。
> 质检时逐项核对，任何一项不通过都需记录。

### 核心检查项（任一不通过 = FAIL）

- [ ] **品牌名称正确**
  - 图中出现的品牌名称与产品事实一致
  - 拼写完全正确，无错字漏字
  - 示例：产品事实为 "XYZHOME"，图中不得出现 "XYZ HOME" 或 "XYZHome"

- [ ] **产品类型正确**
  - 图中展示的产品类型与产品事实一致
  - 示例：产品事实为"窗帘 curtain panel"，图中不得展示成"卷帘 roller shade"

- [ ] **颜色/图案与实物一致**
  - 主色调与参考图一致，无明显偏色
  - 图案类型、位置、密度与实物一致
  - 示例：深灰色产品不得偏蓝或偏紫；有条纹的产品不得变成纯色

- [ ] **尺寸/比例正确**
  - 产品的长宽比例与实物一致
  - 标注的尺寸数值与产品事实一致
  - 示例：52" × 108" 的窗帘在图中应呈现窄长的比例，不得变为正方形

- [ ] **包装数量正确**
  - 图中展示的产品数量与包装内容一致
  - 示例：包装含 2 片窗帘，图中应展示 2 片，不得多展示或少展示

- [ ] **结构正确**
  - 顶部结构类型与产品事实一致（grommet top / rod pocket / tab top 等）
  - 控制面板按钮数量、排列方式与实物一致
  - 孔眼/配件的材质、颜色、光泽与实物一致
  - 示例：grommet top 8 孔银色拉丝，不得变成 rod pocket 或金色亮面

- [ ] **配件正确**
  - 产品事实中"包含"的配件在图中出现（如需要展示）
  - 产品事实中"不包含"的配件在图中不出现
  - 示例：不包含窗帘杆和安装螺丝，图中不得出现这些物品

### 辅助检查项（不通过 = CONDITIONAL）

- [ ] **产品角度合理**
  - 展示角度能清楚呈现产品关键特征
  - 不因角度问题导致关键结构被遮挡

- [ ] **产品完整性**
  - 产品在图中完整呈现，无明显截断（特写图除外）
  - 特写图应聚焦于关键特征区域

- [ ] **产品质感呈现**
  - 表面光泽类型（哑光/拉丝/高光）与实物一致
  - 材质质感（金属/塑料/布艺）与实物一致

## 3.5 可量化细节质检清单

> 以下清单基于 `references/product_fidelity_lock.md` 中的保真锁定卡。
> 这是产品事实质检的深化版本，专门检查 AI 容易"自由发挥"的可量化细节。
> 任一不通过 = FAIL（可量化细节错误等同于产品事实错误）。

### 可数细节检查

- [ ] **孔眼/纽扣/口袋等数量与保真锁定卡一致**
  - 逐一清点生成图中的可数部件
  - 与保真锁定卡中的"每单位数量"和"总数"对比
  - 示例：锁定卡要求 8 grommets per panel, total 16，图中必须恰好 16 个
  - 多1个或少1个都是 FAIL

- [ ] **配件数量与保真锁定卡一致**
  - 图中出现的配件数量与包含清单一致
  - 不包含的配件不出现在图中

### 结构方式检查

- [ ] **悬挂/开合/安装方式与保真锁定卡一致**
  - 检查产品顶部的结构类型（如 grommet top / rod pocket）
  - 检查开合方式（如顶部掀盖 / 前门式）
  - 示例：锁定卡要求 grommet top，图中不得出现 rod pocket
  - 出现替代结构 = FAIL

- [ ] **未出现被 NOT 排除的错误结构**
  - 对照保真锁定卡中的"需排除的错误结构"列表
  - 确认这些错误结构均未出现

### 尺寸比例检查

- [ ] **产品长宽比例与保真锁定卡一致**
  - 生成图中产品的长宽比与底图/锁定卡一致
  - 示例：锁定卡要求 height:width ≈ 3:1，图中不得变为 2:1 或 4:1

- [ ] **未出现明显变长/变短/变宽/变窄**
  - 窗帘类产品特别检查长度是否与锁定卡描述一致
  - 示例：锁定卡要求"reaches floor"，图中不得拖地或过短

### 材质光泽检查

- [ ] **金属表面光泽类型与保真锁定卡一致**
  - 检查光泽是否为锁定卡指定的类型（brushed satin / mirror polished / matte）
  - 示例：锁定卡要求 brushed satin，图中不得出现 mirror polished
  - 光泽类型不符 = FAIL

- [ ] **颜色与保真锁定卡一致，无近似色偏差**
  - 对照锁定卡中的颜色描述和排除的近似色
  - 示例：锁定卡要求 pure white，图中不得偏 cream 或 ivory

### 混合模式额外检查（如使用混合模式）

- [ ] **贴入的真实产品局部与底图完全一致**
  - 检查贴入区域的产品细节是否直接来自底图
  - 贴入区域的孔眼/按钮等数量必须与底图完全一致

- [ ] **合成区域与背景融合自然**
  - 光照方向一致
  - 阴影合理
  - 无明显的"贴图痕迹"

## 4. 合规质检清单

> 以下清单基于 `references/claims_compliance.md` 中的 115 个敏感词清单。
> 合规检查不通过（出现敏感词）= FAIL，无例外。

### 核心检查项（任一不通过 = FAIL）

- [ ] **无 115 个敏感词**
  - 检查图中所有可见文字（包括产品上的文字、标注文字、背景文字）
  - 逐项对照 `references/claims_compliance.md` 中的完整清单
  - 常见高风险词包括：
    - 绝对化：best, #1, top, leading, premium, perfect
    - 医疗/健康：FDA, cure, heal, medical, clinical, doctor
    - 安全认证：UL, ETL, CE, RoHS（未经证实的不得出现）
    - 环保：eco-friendly, green, sustainable, organic
    - 功效：guaranteed, proven, tested, certified
  - 示例：图中文字不得出现 "BEST SELLER" 或 "#1 CHOICE"

- [ ] **无绝对化用语**
  - 不得出现最高级、唯一性表达
  - 常见需排除：best, #1, top, number one, leading, first, only, ultimate
  - 示例：不得标注 "THE BEST CURTAINS" 或 "NO.1 CHOICE"

- [ ] **无未经证实的功能声明**
  - 所有功能声明必须有证据文件支持
  - 未经证实的声明不得出现在图中文字或视觉表达中
  - 常见需排除（无证据时）：energy saving, money saving, 100% blackout（无测试报告时）
  - 示例：没有遮光测试报告，不得标注 "100% BLACKOUT"（可改为 "ROOM DARKENING"）

- [ ] **无竞品品牌**
  - 图中不得出现任何竞品品牌名称或 Logo
  - 检查产品上的文字、背景中的文字、场景中出现的任何品牌标识
  - 示例：自有品牌 XYZHOME 的图中不得出现其他窗帘品牌名称

### 辅助检查项（不通过 = CONDITIONAL）

- [ ] **文字内容合理**
  - 图中文字不产生歧义
  - 不暗示未经证实的产品优势

- [ ] **无误导性视觉表达**
  - 场景图不暗示产品不具备的功能
  - 示例：非防水产品不得在雨天场景中暗示防水

## 5. 技术质检清单

> 以下清单检查图片的技术质量。部分问题可修复（CONDITIONAL），部分不可修复（FAIL）。

### 核心检查项（不可修复 = FAIL）

- [ ] **图片尺寸正确**
  - 图片尺寸符合计划要求
  - Listing 图：1600 × 1600 px（默认，以 Seller Central 要求为准）
  - A+ 桌面版：1464 × 600 px 或 1940 × 1200 px
  - A+ 移动版：1600 × 1200 px
  - 如尺寸错误且无法通过裁切/缩放修复 → FAIL
  - 如可通过裁切/缩放修复 → CONDITIONAL

- [ ] **无拉伸变形**
  - 产品形状无异常拉伸、挤压、扭曲
  - 圆形部件仍为圆形，不得变椭圆
  - 严重变形 → FAIL
  - 轻微变形可接受 → PASS（但建议修复）

### 辅助检查项（可修复 = CONDITIONAL）

- [ ] **文字可读（≥ 36px）**
  - 所有文字标注在图片原始尺寸下字号 ≥ 36px
  - 文字在缩放到展示尺寸后仍可清晰阅读
  - 字号不足或模糊 → CONDITIONAL（可叠加文字层修复）

- [ ] **安全区内无裁切**
  - 亚马逊图片安全区：距边缘 5% 以内不得有关键内容
  - 产品主体、文字标注、品牌信息不得被裁切
  - 安全区内被裁切 → CONDITIONAL（可重新裁切或调整构图）

- [ ] **文字拼写正确**
  - 图中所有文字拼写正确
  - AI 生成文字常见的拼写问题：字母颠倒、漏字母、多余字母
  - 拼写错误 → CONDITIONAL（可叠加正确文字层修复）

- [ ] **图片清晰度**
  - 图片无严重模糊、噪点、像素化
  - 产品细节可辨认
  - 轻微模糊 → CONDITIONAL（可尝试增强或重新生成）

- [ ] **背景干净**
  - 白底图背景为纯白（#FFFFFF）或接近纯白
  - 无杂物、无水印、无其他产品干扰
  - 背景不干净 → CONDITIONAL（可抠图更换背景）

- [ ] **色彩一致**
  - 同一套图中各张图的整体色调一致
  - 无明显的色温、亮度差异
  - 轻微不一致 → CONDITIONAL（可调色统一）

## 6. 常见 FAIL 场景

> 以下场景基于实际质检经验总结，每条包含"失败现象"和"修复建议"。

### FAIL 场景 1：品牌名称拼写错误或变形

- **失败现象**：产品事实品牌为 "XYZHOME"，图中显示为 "XYZ HOME"、"XYZHome"、"XYZ-HOME"。
- **严重程度**：FAIL（品牌错误是关键事实错误）
- **修复建议**：
  1. 在 Prompt 的产品不可变层中明确品牌拼写："品牌名称为 XYZHOME（一个词，无空格无连字符）"。
  2. 在 Negative Prompt 中排除错误拼写。
  3. 如果 AI 仍无法准确生成品牌名，考虑后期手动叠加品牌 Logo 层。

### FAIL 场景 2：产品数量与包装不符

- **失败现象**：包装含 2 片窗帘，图中只展示了 1 片；或包装含 6 件，图中展示了 8 件。
- **严重程度**：FAIL（数量错误是关键事实错误）
- **修复建议**：
  1. 在产品不可变层中明确："包装内含 2 件"。
  2. 在视觉表达层中重申："图中展示 2 片窗帘"。
  3. 在 Negative Prompt 中排除错误数量。

### FAIL 场景 3：顶部结构类型错误

- **失败现象**：产品为 grommet top 窗帘，场景图中顶部呈现为 rod pocket。
- **严重程度**：FAIL（结构错误是关键事实错误）
- **修复建议**：
  1. 在产品不可变层中明确顶部结构类型。
  2. 在视觉表达层中重申："窗帘通过 grommet top 孔眼穿过金属杆悬挂"。
  3. 在 Negative Prompt 中排除：`NOT rod pocket, NOT tab top`。
  4. 详见 `references/prompt_template.md` 第 5 节教训 4。

### FAIL 场景 4：出现敏感词

- **失败现象**：图中文字出现 "BEST"、"#1"、"FDA approved" 等 115 个敏感词之一。
- **严重程度**：FAIL（合规违规，无例外）
- **修复建议**：
  1. 修改 Prompt，删除所有敏感词。
  2. 替换为合规表达（如 "BEST" → 删除或改为中性描述）。
  3. 重新生成图片。
  4. 质检时对照 `references/claims_compliance.md` 完整清单逐词检查。

### FAIL 场景 5：控制面板按钮数量错误

- **失败现象**：真实产品有 4 个按钮，图中有 3 个、5 个或 6 个。
- **严重程度**：FAIL（结构错误是关键事实错误）
- **修复建议**：
  1. 在产品不可变层中明确："控制面板含 4 个按钮"。
  2. 在视觉表达层中重申："清晰显示 4 个按钮"。
  3. 在 Negative Prompt 中排除错误数量：`NOT 3 buttons, NOT 5 buttons`。
  4. 详见 `references/prompt_template.md` 第 5 节教训 2。

### FAIL 场景 6：出现不包含的配件

- **失败现象**：产品不包含安装螺丝和窗帘杆，但图中出现了这些物品。
- **严重程度**：FAIL（配件错误是关键事实错误）
- **修复建议**：
  1. 在产品不可变层中明确"不包含"清单。
  2. 在 Negative Prompt 中排除：`NOT screws, NOT curtain rod, NOT hardware`。
  3. 质检时对照"包含/不包含"清单逐项检查。

### CONDITIONAL 场景 1：文字拼写错误

- **现象**：标注 "BLACKOUT" 生成成了 "BLAKOUT"；"PANELS" 生成成了 "PNELS"。
- **严重程度**：CONDITIONAL（可修复）
- **修复建议**：
  1. 后期在图片上手动叠加正确的文字层。
  2. 或简化文字为纯数字（数字几乎不出错）。
  3. 详见 `references/prompt_template.md` 第 4 节文字标注规范。

### CONDITIONAL 场景 2：轻微偏色

- **现象**：深灰色产品偏蓝或偏紫，但整体色调可辨认。
- **严重程度**：CONDITIONAL（可修复）
- **修复建议**：
  1. 后期调色，降低蓝色/紫色色偏。
  2. 在 Prompt 中增加颜色约束："dark gray (not blue, not purple)"。
  3. 在 Negative Prompt 中排除偏色：`NOT blue tint, NOT purple tint`。

### CONDITIONAL 场景 3：文字字号不足

- **现象**：文字标注在原始尺寸下小于 36px，缩放后难以阅读。
- **严重程度**：CONDITIONAL（可修复）
- **修复建议**：
  1. 后期重新叠加更大字号的文字层。
  2. 在 Prompt 中明确字号要求："文字至少 48px"。

### CONDITIONAL 场景 4：安全区内被裁切

- **现象**：产品边缘或文字标注距图片边缘不足 5%，有被裁切风险。
- **严重程度**：CONDITIONAL（可修复）
- **修复建议**：
  1. 重新裁切图片，确保安全区内无关键内容。
  2. 或调整构图，将关键内容移至安全区内。

## 8. A+ 内容质检清单

> 以下清单是 A+ 内容图片（桌面版 + 移动版）的专属质检标准。
> A+ 图片的通用质检（产品事实、合规、保真细节）与 Listing 图一致，参见第 3-5 节。
> 本节仅列出 A+ 专属的额外检查项。
> 详见 `references/aplus_content_guide.md`。

### 8.1 桌面版专属检查项（aplus-desktop-01 至 05）

#### 核心检查项（不可修复 = FAIL）

- [ ] **尺寸正确**
  - 图片尺寸为 1464 × 600 px 或 1940 × 1200 px
  - 横版比例约 2.44:1 或约 1.62:1
  - 尺寸错误且无法裁切修复 → FAIL

- [ ] **横版构图**
  - 图片为横向构图，左右分区布局
  - 不得为竖版或正方形
  - 出现竖版构图 → FAIL

- [ ] **文字字号 ≥ 36px**
  - 所有文字标注在原始尺寸下字号 ≥ 36px
  - 字号不足 → CONDITIONAL（可叠加文字层修复）

#### 辅助检查项（可修复 = CONDITIONAL）

- [ ] **左右分区清晰**
  - 桌面版采用左右分区构图（产品+信息/场景）
  - 分区比例合理，不偏挤
  - 分区混乱 → CONDITIONAL

- [ ] **信息密度合理**
  - 桌面版可容纳较多信息，但不堆砌
  - 每张图聚焦 1-2 个核心信息点
  - 信息过载 → CONDITIONAL

- [ ] **色块/渐变过渡自然**
  - 桌面版常使用色块或渐变作为分区过渡
  - 过渡自然，不突兀
  - 过渡生硬 → CONDITIONAL

### 8.2 移动版专属检查项（aplus-mobile-01 至 05）

#### 核心检查项（不可修复 = FAIL）

- [ ] **尺寸正确**
  - 图片尺寸为 1600 × 1200 px
  - 竖版比例 4:3
  - 尺寸错误且无法裁切修复 → FAIL

- [ ] **竖版构图**
  - 图片为纵向构图，上下分区布局
  - 不得为横版或正方形
  - 出现横版构图 → FAIL

- [ ] **独立构图验证（Hard Rule 9）**
  - 移动版必须为独立构图，不得是桌面版的缩放或裁切版本
  - 检查方法：对比同序号的桌面版，确认构图方向、产品位置、文字布局均不同
  - 移动版为桌面版缩放 → **FAIL**（违反 Hard Rule 9）

- [ ] **文字字号 ≥ 48px**
  - 移动版文字字号必须 ≥ 48px（比桌面版更大）
  - 小屏可读性要求更高
  - 字号不足 → CONDITIONAL（可叠加文字层修复）

#### 辅助检查项（可修复 = CONDITIONAL）

- [ ] **上下分区清晰**
  - 移动版采用上下分区构图（产品+信息）
  - 分区比例合理，上方产品为主
  - 分区混乱 → CONDITIONAL

- [ ] **信息精简**
  - 移动版信息量应少于桌面版
  - 每张图聚焦 1 个核心信息点
  - 信息过多 → CONDITIONAL

- [ ] **文字短词为主**
  - 移动版文字以短词和数字为主
  - 避免长句（手机屏幕宽度有限）
  - 文字过长 → CONDITIONAL

### 8.3 A+ 通用检查项（桌面版 + 移动版）

> 以下检查项适用于所有 A+ 图片，与 Listing 图的标准一致。

- [ ] **产品保真与 Listing 一致**
  - A+ 图片中的产品形状、颜色、数量、结构与 Listing 图一致
  - 对照保真锁定卡检查可量化细节
  - 不一致 → FAIL

- [ ] **产品锁段落已包含**
  - A+ 图片的 Prompt 中必须包含产品锁段落
  - 与 Listing 图共用同一产品锁段落
  - 未包含 → FAIL

- [ ] **合规扫描通过**
  - A+ 图片中所有文字通过 115 敏感词扫描
  - 无绝对化用语、无虚假声明、无竞品品牌
  - 违规 → FAIL

- [ ] **无水印**
  - A+ 图片不得包含任何水印
  - 包括 AI 生成水印、品牌水印等
  - 有水印 → CONDITIONAL（可去除）

- [ ] **Desktop 与 Mobile 主题对应**
  - 同序号的桌面版和移动版应展示相同主题
  - 画面内容可以不同（构图独立），但核心信息一致
  - 主题不对应 → CONDITIONAL

### 8.4 A+ 常见 FAIL 场景

#### FAIL 场景 7：移动版缩放桌面版

- **失败现象**：移动版图片明显是桌面版图片的缩放或裁切版本，而非独立构图。
- **严重程度**：FAIL（违反 Hard Rule 9）
- **修复建议**：
  1. 不得缩放桌面版，必须为移动版独立编写 Prompt。
  2. 移动版采用纵向构图（上下分区），桌面版采用横向构图（左右分区）。
  3. 详见 `references/aplus_content_guide.md` 第 6 节构图差异对照。

#### FAIL 场景 8：A+ 尺寸错误

- **失败现象**：桌面版尺寸非 1464×600 或 1940×1200，或移动版尺寸非 1600×1200。
- **严重程度**：FAIL（尺寸错误，如无法裁切修复）
- **修复建议**：
  1. 检查 Prompt 中的尺寸声明是否正确。
  2. 桌面版 Prompt 必须包含 "horizontal 1464x600" 或 "horizontal 1940x1200"。
  3. 移动版 Prompt 必须包含 "vertical 1600x1200"。

#### FAIL 场景 9：A+ 产品保真不一致

- **失败现象**：A+ 图片中的产品与 Listing 图中的产品不一致（如颜色偏差、结构改变）。
- **严重程度**：FAIL（产品事实错误）
- **修复建议**：
  1. 确认 A+ Prompt 中包含了与 Listing 图相同的产品不可变层和产品锁段落。
  2. A+ 图不重新锁定产品事实，直接复用 Listing 阶段已锁定的内容。
  3. 详见 `references/aplus_content_guide.md` 第 3 节。
