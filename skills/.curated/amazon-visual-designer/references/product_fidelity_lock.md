# 产品保真锁定卡（Product Fidelity Lock）

> 本文件是 Amazon Visual Designer 的核心参考文件之一。
> 当从产品底图提取特征时加载本文件，用于锁定可量化细节。
>
> 设计原则：
> 1. 可量化优先 —— 能数的一定数，能量的一定量，能用精确术语的一定用术语。
> 2. 锁定即不可变 —— 一旦用户确认，所有后续步骤不得改变。
> 3. 四类细节全覆盖 —— 可数细节、结构方式、尺寸比例、材质光泽。

## 1. 为什么需要保真锁定卡

### 问题根源

AI 文生图模型是"创造性重绘"，不是"精确复制"。每次生成都是独立解读提示词，无法保证产品细节的精确一致性。具体表现为：

| 失控类型 | 示例 | 后果 |
|----------|------|------|
| 可数细节变化 | 孔眼16个→18个、按钮6个→4个 | 亚马逊判定为不同产品 |
| 结构方式替换 | grommet top→rod pocket | 产品事实不符 |
| 尺寸比例偏移 | 108"→变长/变短 | 误导买家，引发退货 |
| 材质光泽改变 | 拉丝哑光→镜面高光 | 产品质感不一致 |

### 保真锁定卡的作用

保真锁定卡是在产品事实锁定之后、图片生成之前，额外锁定一层"可量化细节"，确保 AI 在生成时有一个精确的约束基准，在质检时有一个可逐项核对的检查清单。

## 2. 四类可量化细节

### 2.1 可数细节

产品中可以用数字精确计数的部件。

| 产品类型 | 常见可数细节 | 示例 |
|----------|-------------|------|
| 窗帘 | 孔眼数量 | 8 grommets per panel, 2 panels, total 16 |
| 家电 | 按钮数量 | 6 buttons in 2 rows × 3 columns |
| 收纳架 | 层数/格数 | 5 tiers, 3 compartments per tier |
| 餐具套装 | 件数 | 6 forks, 6 knives, 6 spoons = 18 pieces |
| 灯具 | 灯头数量 | 3 light bulbs, E26 base |

提取规则：
- 必须写明"每单位数量"和"总数"
- 如有排列方式，必须描述（如"2行3列"）
- 质检时必须逐一清点

### 2.2 结构方式

产品的安装、悬挂、开合、连接等结构类型的精确名称。

| 产品类型 | 常见结构方式 | 需要排除的错误结构 |
|----------|-------------|-------------------|
| 窗帘 | grommet top / rod pocket / tab top / back tab | NOT rod pocket（当需要grommet时） |
| 家电 | 顶部掀盖 / 前门式 / 抽屉式 | NOT front door（当需要top lid时） |
| 收纳架 | 可折叠 / 固定式 / 可堆叠 | NOT folding（当为固定式时） |
| 灯具 | 吸顶 / 吊挂 / 壁挂 / 落地 | NOT pendant（当为flush mount时） |

提取规则：
- 必须使用行业通用术语
- 必须列出需要排除的常见错误结构
- 质检时必须验证结构类型未被替换

### 2.3 尺寸比例

产品的长宽高比例和视觉长度描述。

| 维度 | 描述方式 | 示例 |
|------|----------|------|
| 整体比例 | 长:宽比例 | tall vertical slender (height:width ≈ 3:1) |
| 视觉长度 | 相对参照物 | curtain reaches floor, NOT dragging |
| 部件比例 | 部件相对于整体 | knob is 1/5 of lid height |
| 尺寸标注 | 精确数值 | 108 × 108 inches, 54 inches per panel |

提取规则：
- 比例必须量化（如"3:1"而非"很长"）
- 视觉长度用参照物描述（如"及地""及腰"）
- 质检时对比生成图与底图的比例

### 2.4 材质光泽

产品表面的材质和光泽类型的精确描述。

| 维度 | 常见选项 | 示例 |
|------|----------|------|
| 金属光泽 | brushed satin / mirror polished / matte / chrome | brushed satin finish (NOT mirror) |
| 塑料质感 | glossy / matte / textured / smooth | matte ABS plastic |
| 布料质感 | sheer / room darkening / blackout / heavy weight | medium weight, matte |
| 颜色精度 | 英文色名 + 排除近似色 | mint green (NOT teal, NOT sage) |

提取规则：
- 光泽类型必须用精确术语
- 必须列出容易混淆的近似光泽并排除
- 颜色必须用英文色名，并排除近似色
- 质检时对比生成图与底图的光泽和颜色

## 3. 提取流程

```
步骤1：读取产品底图
  ↓
步骤2：提取可数细节
  → 逐一清点可见的孔眼、按钮、配件等
  → 记录每单位数量和总数
  → 记录排列方式
  ↓
步骤3：识别结构方式
  → 判断安装/悬挂/开合类型
  → 用行业术语命名
  → 列出需要排除的常见错误结构
  ↓
步骤4：描述尺寸比例
  → 从底图判断整体比例
  → 如有尺寸标注则提取
  → 用参照物描述视觉长度
  ↓
步骤5：判断材质光泽
  → 从图片光泽和纹理推断
  → 用精确术语描述
  → 列出容易混淆的近似光泽并排除
  ↓
步骤6：生成保真锁定卡
  → 填写第4节模板
  → 生成第5节产品锁段落
  ↓
步骤7：向用户确认
  → 展示保真锁定卡
  → 用户确认或修正
  → 锁定，后续不可变
```

## 5. 产品锁段落（Prompt 用）

产品锁段落是保真锁定卡的 Prompt 版本，每张图的 Prompt 中必须原样包含此段落。

### 使用规则

1. **原样复制**：产品锁段落一旦从保真锁定卡生成，所有 Listing 图和 A+ 图必须原样包含，不得修改。
2. **位置固定**：放在产品不可变层之后、视觉表达层之前。
3. **NOT 约束**：数量必须列出 `NOT 数量-1, NOT 数量+1` 等排除项，因为 AI 不会数数，需要明确排除错误数量。
4. **与保真锁定卡一致**：产品锁内容必须与保真锁定卡完全一致，不得有出入。
5. **A+ 图同样适用**：A+ 桌面版和移动版图片的 Prompt 中也必须包含此段落。A+ 图不重新生成产品锁，直接复用 Listing 阶段已锁定的内容。详见 `references/aplus_content_guide.md` 第 3 节。

### 模板

```
【产品锁 — 不可更改，每张图必须包含】
- EXACTLY {数量} {部件} per {单位}, total {总数}
  (NOT {数量-1}, NOT {数量+1}, NOT {数量+2})
- {结构名称} ONLY
  (NOT {错误结构1}, NOT {错误结构2}, NOT {错误结构3})
- {尺寸描述}
  (NOT {错误尺寸描述})
- {材质描述}, {光泽类型}
  (NOT {错误光泽1}, NOT {错误光泽2})
- {颜色描述}
  (NOT {近似色1}, NOT {近似色2})
【产品锁 — 结束】
```

### 示例：窗帘产品

```
【产品锁 — 不可更改，每张图必须包含】
- EXACTLY 8 grommets per panel, total 16 grommets (2 panels)
  (NOT 6, NOT 7, NOT 9, NOT 10, NOT 12 grommets)
- GROMMET TOP ONLY
  (NOT rod pocket, NOT tab top, NOT back tab, NOT clip rings)
- 108 inches long, reaches floor
  (NOT dragging, NOT too short, NOT 84 inches, NOT 96 inches)
- Silver stainless steel grommets, brushed satin finish
  (NOT mirror polished, NOT chrome, NOT gold, NOT bronze)
- Pure white fabric
  (NOT cream, NOT ivory, NOT gray)
【产品锁 — 结束】
```

### 示例：家电产品

```
【产品锁 — 不可更改，每张图必须包含】
- EXACTLY 6 buttons, 2 rows × 3 columns layout
  (NOT 3, NOT 4, NOT 5, NOT 8 buttons)
- TOP LID opens upward (clamshell hinge)
  (NOT front door, NOT drawer, NOT removable lid)
- Tall vertical slender shape, height:width ≈ 3:1
  (NOT short, NOT wide, NOT square)
- White plastic body with mint green accents, matte finish
  (NOT glossy, NOT all white, NOT gray)
- "momcozy" logo inside control panel, mint green text
  (NOT on body, NOT different spelling)
【产品锁 — 结束】
```

## 6. 精度模式选择

在 Phase 1 规划阶段，让用户选择生图精度模式。
精度模式对 Listing 图和 A+ 图统一生效，用户只需选择一次。

### 6.1 两种模式对比

| 维度 | AI生图模式 | 混合模式 |
|------|-----------|----------|
| 适用产品 | 简单产品、结构单一 | 复杂产品、可数细节多 |
| 适用场景 | 对细节精度要求不高 | 对细节精度要求高 |
| 生成方式 | AI 全权生成 | AI生成背景 + 用户PS贴入真实产品局部 |
| 孔眼/按钮等细节 | AI 尝试生成，可能偏差 | 用户从底图截取真实局部贴入 |
| 精度保证 | 中等（依赖提示词约束+质检返工） | 高（真实产品局部保证一致性） |
| 效率 | 高（全自动） | 中等（需要手动PS） |
| 返工概率 | 较高 | 较低 |

### 6.2 模式选择建议

| 产品特征 | 推荐模式 |
|----------|----------|
| 有精确数量的可数细节（孔眼、按钮等） | 混合模式 |
| 结构方式容易被AI替换（如grommet vs rod pocket） | 混合模式 |
| 简单结构、无复杂细节 | AI生图模式 |
| 纯色无配件的产品 | AI生图模式 |
| 品牌Logo必须精确 | 混合模式 |
| 场景图为主、产品细节不重要 | AI生图模式 |

### 6.3 混合模式工作流

```
步骤1：AI 生成场景背景（不含产品关键细节区域）
  → Prompt 中标注："leave blank space for product overlay"
  → 生成带留白的场景图
  ↓
步骤2：输出合成指引（hybrid_guide_[image_id].md）
  → 指明哪个区域需要贴入真实产品局部
  → 提供底图中对应区域的截取建议
  → 提供PS合成参数（如缩放比例、旋转角度、光照匹配）
  ↓
步骤3：用户在PS中合成
  → 从底图截取真实产品局部
  → 贴入AI生成的场景图中
  → 调整光照、阴影、比例
  ↓
步骤4：合成后重新质检
  → 检查合成区域与背景的融合度
  → 检查产品细节是否与底图一致
```

### 6.4 混合模式指引模板

```markdown
# 混合模式合成指引 - {image_id}

- 文件：{AI生成的场景背景图}
- 已生成区域：{描述}
- 留白区域：{描述位置和大小}

- [ ] 阴影合理
```

## 7. 诚实降级协议

### 7.1 触发条件

当 AI 生图返工 3 次后，仍无法通过保真质检时，启动诚实降级协议。

### 7.2 降级流程

```
返工第1次：调整Prompt → 重新生成 → 质检
  ↓ （不通过）
返工第2次：加强约束词 → 重新生成 → 质检
  ↓ （不通过）
返工第3次：最后一次尝试 → 重新生成 → 质检
  ↓ （仍不通过）
启动诚实降级协议
  ↓
步骤1：标记为 CONDITIONAL
  ↓
步骤2：记录具体偏差
  → 哪些细节与保真锁定卡不一致
  → 偏差程度（轻微/中等/严重）
  → 建议的修复方式
  ↓
步骤3：告知用户
  → 展示生成图与底图的对比
  → 列出具体偏差项
  → 建议手动修正方式（PS贴图/局部重绘）
  → 或建议切换到混合模式
  ↓
步骤4：等待用户决策
  → 用户手动修正 → 修正后重新质检
  → 用户切换混合模式 → 按混合模式重新生成
  → 用户接受偏差 → 标注"已知情接受"，记录在QA报告中
```

### 7.3 降级报告模板

```markdown
## 降级报告 - {image_id}

### 返工历史
| 次数 | 调整内容 | 质检结果 | 偏差项 |
|------|----------|----------|--------|
| 1 | {调整描述} | {FAIL/CONDITIONAL} | {偏差描述} |
| 2 | {调整描述} | {FAIL/CONDITIONAL} | {偏差描述} |
| 3 | {调整描述} | {FAIL/CONDITIONAL} | {偏差描述} |

### 当前偏差
| 保真锁定卡要求 | 生成图实际呈现 | 偏差程度 |
|---------------|---------------|----------|
| {如：8 grommets per panel} | {如：9 grommets} | {中等} |
| {如：grommet top} | {如：grommet top（正确）} | {无} |

### 建议修复方式
1. {如：手动删除多余的1个孔眼}
2. {如：切换到混合模式，贴入真实孔眼区域}

### 用户决策
- [ ] 手动修正（用户自行PS）
- [ ] 切换混合模式重新生成
- [ ] 接受偏差（已知情）
```

### 7.4 禁止事项

- 不得假装通过了质检
- 不得隐瞒偏差
- 不得自动接受偏差（必须用户明确同意）
- 不得跳过降级报告
