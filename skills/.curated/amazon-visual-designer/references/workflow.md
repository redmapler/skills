## Phase 0: 输入验证（双通道输入 + 自动提取）

> Phase 0 是整个工作流的"地基"。
> Skill 支持双通道输入，用户只需提供产品底图（白底图/实拍图），Skill 自动从图片中提取产品特征。
> 提取后向用户确认，补充底图无法判断的信息（如尺寸、数量等）。
> 用户确认后锁定产品事实，进入 Phase 1。

### 0.0 双通道输入

Skill 支持两种输入方式，自动识别：

- **方式A：聊天窗口直接传图** — 用户在聊天中直接上传1张或多张产品底图
- **方式B：提供文件夹路径** — 用户提供产品文件夹路径，Skill 读取文件夹中的图片

两种方式的后续流程完全一致：读取图片 → 提取特征 → 确认事实。

> 注意：无论用户使用哪种方式输入，Skill 都应自动识别并统一进入后续流程，不需要用户额外指定输入方式。

### 0.0.1 用户需求快速收集

> 在读取底图之前，先快速了解用户的核心需求，以便后续场景设计和色调选择更贴合用户期望。
> 如果用户在触发时已通过推荐模板提供了信息，则跳过此步骤。

如果用户未提供以下信息，主动询问（3-5 个关键问题，一次性问完，避免反复追问）：

1. **目标受众**：谁在使用这个产品？（如：家庭主妇、户外爱好者、企业采购）
2. **品牌调性**：产品定位是？（如：高端精致、性价比实用、年轻潮流）
3. **视觉风格偏好**：喜欢什么风格？（如：简约白底、生活场景实拍、科技感、温馨家居）
4. **产品卖点（一句话）**：最想突出的卖点是什么？
5. **是否需要 A+ 内容**：是 / 否

> 以上信息不是必须全部回答，用户回答越多，生成的场景和色调越贴合预期。不回答则按通用策略执行。

### 0.1 读取产品底图

#### 0.1.1 输入要求

用户上传1张或多张产品底图（JPG/PNG/WEBP格式）。底图应满足：
- 画面清晰，能看清产品形状、颜色、图案
- 产品主体完整可见
- 背景为白色或纯色（首选白底图）
- 如有多张底图（正面、侧面、细节等），全部读取

#### 0.1.2 底图不满足要求时的处理

| 问题 | 处理方式 |
|------|----------|
| 图片模糊，看不清细节 | 告警，要求用户补充清晰图 |
| 产品被遮挡，主体不完整 | 告警，要求用户补充完整图 |
| 背景杂乱 | 告警，建议补充白底图，但不阻断流程 |
| 图片格式不支持 | 要求用户转换为 JPG/PNG |

> 注意：宁可停下来问用户，也不要靠猜去填补缺失信息。

### 0.1.3 底图覆盖度评估（Hard Rule 10）

> 读取底图后，必须评估底图的覆盖度。底图不足会直接导致后续生成图与真实产品出现细节偏差。
> 评估不通过时，必须在第一时间告知用户，要求补充。用户未补充或未明确确认继续前，不得进入 0.2。

#### 评估维度

| 评估维度 | 要求 | 不足时的处理 |
|----------|------|-------------|
| **正面视角** | 至少1张清晰正面图（产品主体完整可见） | 必须补充，阻断流程 |
| **侧面/背面视角** | 至少1张侧面或背面图（展示产品深度和背面结构） | 告警，要求补充；用户确认可继续则标注为"视角缺失，场景图可能偏差" |
| **关键细节特写** | 至少1张关键细节图（如孔眼、按钮、纹理、接口等） | 告警，要求补充；用户确认可继续则标注为"细节缺失，保真度降低" |
| **清晰度** | 所有图片清晰可辨，无严重模糊 | 必须补充，阻断流程 |
| **数量** | 至少3张不同角度/细节的底图 | 不足3张时告警，要求补充 |

#### 评估结果展示

向用户展示评估结果：

```
【底图覆盖度评估】
- 底图数量：{N} 张
- 正面视角：✅ 有 / ❌ 缺失
- 侧面/背面视角：✅ 有 / ❌ 缺失
- 关键细节特写：✅ 有 / ❌ 缺失
- 清晰度：✅ 合格 / ❌ 不合格

评估结论：
- ✅ 覆盖度充足，可进入后续流程
- ⚠️ 覆盖度不足，缺失以下视角：
  - {缺失项1}
  - {缺失项2}
  请补充以上底图，或回复"继续生成"确认在底图不足的情况下继续。
  注意：底图不足可能导致生成图与真实产品出现细节偏差。
```

#### 用户确认继续时的处理

如果用户明确回复"继续生成"（或类似确认），则：
1. 记录缺失的视角和细节
2. 在后续生成 Prompt 中增加约束（如"侧背面视角缺失，避免生成需要背面细节的场景"）
3. 在 QA 报告中标注"底图覆盖度不足，用户确认继续"
4. 不阻断流程，但质保标准不降低

> 关键原则：底图不足时，宁可让用户补图，也不要在信息不足的情况下盲目生成。
> 生成出来的图片与真实产品差距大，返工成本远高于补图成本。

### 0.2 从底图自动提取产品特征

读取底图后，自动识别以下产品事实：

| 字段 | 提取方式 | 底图无法判断时 |
|------|----------|----------------|
| 品牌 | 从产品上的Logo/文字识别 | 询问用户 |
| 产品类型 | 从产品形状和结构判断 | 询问用户 |
| 颜色/图案 | 从图片直接识别 | 正常提取 |
| 形状结构 | 从图片判断（如圆顶/方形、孔眼类型等） | 正常提取 |
| 材质质感 | 从图片光泽和纹理推断（标注为"推测"） | 询问用户确认 |
| 尺寸 | 如底图有尺寸标注则提取 | **必须询问用户** |
| 包装数量 | 无法从单张底图判断 | **必须询问用户** |
| 包含/不包含 | 无法从底图判断 | **必须询问用户** |

#### 0.2.1 特征提取要点

- **形状结构**：必须精确描述。如"竖高修长型""圆顶弧形""6个按钮2行3列布局"等
- **颜色**：使用准确的英文色名（如 mint green、brushed silver、pure white）
- **材质质感**：区分"拉丝哑光"和"高抛光镜面"等不同质感
- **关键细节**：如孔眼类型（grommet top / rod pocket）、控制面板布局、配件形态等

#### 0.2.2 提取结果展示

将提取的产品事实以表格形式展示给用户：

```
| 字段 | 提取结果 | 来源 | 置信度 |
|------|----------|------|--------|
| 品牌 | Momcozy | 底图Logo识别 | 高 |
| 产品类型 | 奶瓶清洗机 | 底图形状判断 | 高 |
| 颜色 | 白色+薄荷绿 | 底图直接识别 | 高 |
| 材质 | 塑料+金属（推测） | 底图纹理推断 | 中（需确认） |
| 尺寸 | 未知 | 底图无标注 | 需用户提供 |
| 数量 | 未知 | 无法判断 | 需用户提供 |
```

### 0.2.3 提取可量化细节（保真锁定卡）

> AI 文生图模型是"创造性重绘"，不是"精确复制"。每次生成都是独立解读提示词，无法保证产品细节的精确一致性。
> 为了在图片生成前锁定一层"可量化细节"，确保 AI 在生成时有一个精确的约束基准，在质检时有一个可逐项核对的检查清单，
> 必须在此步骤中提取可量化细节，填写保真锁定卡。详见 `references/product_fidelity_lock.md`。

在此步骤中，除了提取基本产品特征（0.2），还必须提取可量化细节，填写保真锁定卡。

#### 0.2.3.1 提取四类可量化细节

**1. 可数细节**
- 产品中可以用数字精确计数的部件：孔眼、按钮、口袋、层架、灯头等
- 提取规则：
  - 必须写明"每单位数量"和"总数"
  - 如有排列方式，必须描述（如"2行3列"）
  - 示例：`8 grommets per panel, 2 panels, total 16`

**2. 结构方式**
- 产品的安装、悬挂、开合、连接等结构类型的精确名称
- 提取规则：
  - 必须使用行业通用术语（如 grommet top / rod pocket / tab top）
  - 必须列出需要排除的常见错误结构
  - 示例：`grommet top (NOT rod pocket, NOT tab top, NOT back tab)`

**3. 尺寸比例**
- 产品的长宽高比例和视觉长度描述
- 提取规则：
  - 比例必须量化（如"3:1"而非"很长"）
  - 视觉长度用参照物描述（如"及地""及腰"）
  - 如有精确尺寸标注则提取
  - 示例：`tall vertical slender, height:width ≈ 3:1, reaches floor`

**4. 材质光泽**
- 产品表面的材质和光泽类型的精确描述
- 提取规则：
  - 光泽类型必须用精确术语（如 brushed satin / mirror polished / matte）
  - 必须列出容易混淆的近似光泽并排除
  - 颜色必须用英文色名，并排除近似色
  - 示例：`brushed satin finish (NOT mirror polished, NOT chrome); mint green (NOT teal, NOT sage)`

#### 0.2.3.2 生成保真锁定卡和产品锁段落

提取四类可量化细节后：
1. 按模板生成保真锁定卡（模板见 `references/product_fidelity_lock.md` 第4节）
2. 从保真锁定卡生成产品锁段落（模板见 `references/product_fidelity_lock.md` 第5节），该段落将在 Phase 2 中作为每张图 Prompt 的必含内容

> 注意：保真锁定卡和产品锁段落必须一并向用户展示确认（见 0.3），确认后锁定，后续不可变。

### 0.3 向用户确认并补充信息

#### 0.3.1 必须询问用户的信息

以下信息通常无法从底图提取，必须向用户询问：

1. **尺寸**：产品的精确尺寸（长×宽×高，带单位）
2. **包装数量**：套装包含几件产品
3. **包含/不包含**：配件清单，特别是不包含的物品
4. **材质确认**：如果从图片推测了材质，需用户确认
5. **品牌确认**：如果从Logo识别了品牌，需用户确认拼写
6. **保真锁定卡确认**：展示保真锁定卡，请用户确认或修正四类可量化细节

> 向用户提问时，以填空模板格式呈现，降低用户输入成本：
> ```
> 请补充以下信息（可直接复制填写）：
> - 产品尺寸：___ × ___ × ___ cm
> - 包装数量：___ 件
> - 包含配件：___、___、___
> - 不包含：___、___
> - 材质：___
> ```

#### 0.3.2 可选询问的信息

1. **产品卖点**：用户希望突出的核心卖点（不使用敏感词）
2. **适用场景**：产品使用的典型场景
3. **竞品参考**：是否有竞品图片可供参考（仅用于研究）
4. **风格偏好**：场景图的风格偏好（如简约北欧、温馨家居等）

#### 0.3.3 确认流程

```
步骤1：展示从底图提取的产品特征表（0.2）
  ↓
步骤2：展示保真锁定卡 + 产品锁段落（0.2.3）
  ↓
步骤3：列出需要用户补充的字段（尺寸、数量等）
  ↓
步骤4：用户确认或修正提取结果和保真锁定卡
  ↓
步骤5：用户补充缺失信息
  ↓
步骤6：锁定产品事实 + 保真锁定卡，进入 Phase 1
```

> 关键原则：产品事实和保真锁定卡不锁定，不进入 Phase 1。这是为了防止策划阶段建立在错误事实之上，也是为了给 AI 生图提供精确的约束基准。

### 0.4 产品底图的后续用途

产品底图在整个流程中有以下用途：
- **主图参考**：作为主图的基础参考，AI 只能增强场景，不能改变产品本身
- **副图参考**：副图中的产品必须与底图一致
- **质检基准**：生成的图片要与底图对比，确保产品事实未被篡改
- **保真锁定卡来源**：从底图提取的可量化细节形成保真锁定卡，作为每张图 Prompt 的产品锁段落
- **混合模式素材**：如用户选择混合模式，底图将作为真实产品局部的截取来源
- **统一描述模板来源**：从底图提取的特征将写入所有图片共用的"产品不可变层"

> 注意：产品底图是"产品不可变层"和"产品锁段落"的核心证据。后续所有图片中产品的形状、颜色、数量、结构、可量化细节都必须与底图一致。

## Phase 1: 规划 + 确认（不生图）

> Phase 1 只输出策略和规划，不生成最终图片。
> 不再生成7个独立 md 文件，而是在聊天中直接展示4项内容，让用户快速确认。
> 用户明确确认后，才进入 Phase 2 执行。
> 用户没有回复确认前，不得生成最终 Prompt 或图片。

### 1.1 展示内容总览

Phase 1 在聊天中直接展示以下4项内容，不再生成独立文件：

| 序号 | 内容 | 作用 |
|------|------|------|
| 1 | 产品特征摘要表 | 从底图提取的所有特征 + 保真锁定卡一览 |
| 2 | 图片规划摘要表 | 由 Skill 根据产品类型规划图片数量和内容（主图 + 细节图 + 场景图 + 信息图 + 套装图） |
| 3 | 精度模式选择 | 让用户选择生图模式（AI生图模式 / 混合模式） |
| 4 | A+ 内容规划摘要表 | A+ 桌面版5张 + 移动版5张的规划（可选） |

展示完毕后显示门禁文案，等用户确认。

### 1.2 产品特征摘要表

#### 目的
将 Phase 0 提取的产品特征和保真锁定卡汇总为一张简洁的摘要表，让用户快速确认产品事实是否准确。

#### 模板结构

```markdown
## 产品特征摘要表

### 基本产品特征

| 字段 | 锁定值 | 来源 | 置信度 |
|------|--------|------|--------|
| 品牌 | [填入或"未提供"] | [底图Logo识别/用户提供] | [高/中] |
| 产品类型 | [填入] | [底图形状判断/用户提供] | [高] |
| 颜色/图案 | [填入，英文色名] | [底图直接识别] | [高] |
| 形状结构 | [填入，精确描述] | [底图判断] | [高] |
| 尺寸 | [填入，带单位] | [用户提供] | [高] |
| 包装数量 | [填入] | [用户提供] | [高] |
| 包含 | [填入清单] | [用户提供] | [高] |
| 不包含 | [填入清单] | [用户提供] | [高] |
| 材质 | [填入] | [底图推断/用户确认] | [中/高] |

### 保真锁定卡一览

#### 可数细节
| 部件名称 | 每单位数量 | 总数 | 排列方式 |
|----------|-----------|------|----------|
| [如：grommets] | [如：8] | [如：16] | [如：均匀分布于顶部] |

#### 结构方式
| 结构类型 | 锁定值 | 需排除的错误结构 |
|----------|--------|-----------------|
| [如：悬挂方式] | [如：grommet top] | [如：NOT rod pocket, NOT tab top] |

#### 尺寸比例
| 维度 | 锁定值 |
|------|--------|
| 整体比例 | [如：height:width ≈ 3:1] |
| 视觉长度 | [如：reaches floor] |

#### 材质光泽
| 部件 | 材质 | 光泽类型 | 需排除的近似光泽 |
|------|------|----------|-----------------|
| [如：主体] | [如：ABS plastic] | [如：matte] | [如：NOT glossy] |
```

#### 填写规则
1. 所有字段必须填写，缺失则填"未知"。
2. 保真锁定卡一览必须与 Phase 0 用户确认后的内容一致。
3. 如有证据等级为 C/D 的字段，需在此表中标注告警。

### 1.3 图片规划摘要表

#### 目的
由 Skill 根据产品类型、可用底图和卖点，规划图片数量和内容。图片数量非固定，而是根据产品复杂度和底图覆盖度灵活决定。场景图的具体场景也由 Skill 设计规划。

#### 规划原则

1. **主图（1张）**：必须有，纯白背景产品主体图
2. **细节/功能图（1-4张）**：根据产品复杂度决定
   - 简单产品（如纯色窗帘）：1-2张（材质特写、结构特写）
   - 复杂产品（如电器、多功能产品）：3-4张（各功能部位、控制面板、配件等）
3. **场景图（1-4张）**：由 Skill 根据产品类型设计具体场景
   - Skill 应根据产品的实际使用场景设计，而非套用固定模板
   - 如窗帘：Patio / Porch / Gazebo / Pergola 等户外场景
   - 如厨房电器：厨房台面 / 餐厅 / 开放式厨房等
   - 场景数量根据产品卖点丰富度决定
4. **尺寸/信息图（1-2张）**：尺寸标注图 + 规格信息图（如适用）
5. **套装内容图（1张）**：展示包装内容（如有多件装）

#### 规划流程

```
步骤1：分析产品类型和复杂度
  → 简单产品 → 少量细节图
  → 复杂产品 → 多量细节图
  ↓
步骤2：分析底图覆盖度（参考 0.1.3 评估结果）
  → 覆盖度高 → 可规划更多细节图
  → 覆盖度低 → 减少需要底图支撑的图片，增加场景图
  ↓
步骤3：设计场景
  → 根据产品实际使用场景设计
  → 考虑产品卖点与场景的匹配
  → 避免重复雷同的场景
  ↓
步骤4：汇总规划
  → 确定总图片数量
  → 列出每张图的主题、画面内容、对应卖点
  → 展示给用户确认
```

### 1.4 精度模式选择

#### 目的
让用户根据产品特征选择生图精度模式。精度模式决定 Phase 2 的执行方式。

#### 1.4.1 两种模式对比

> 注意：本表与 references/product_fidelity_lock.md 第6.1节内容一致，修改时需同步更新。

| 维度 | AI生图模式 | 混合模式 |
|------|-----------|----------|
| 适用产品 | 简单产品、结构单一 | 复杂产品、可数细节多 |
| 适用场景 | 对细节精度要求不高 | 对细节精度要求高 |
| 生成方式 | AI 全权生成 | AI生成背景 + 用户PS贴入真实产品局部 |
| 孔眼/按钮等细节 | AI 尝试生成，可能偏差 | 用户从底图截取真实局部贴入 |
| 精度保证 | 中等（依赖提示词约束+质检返工） | 高（真实产品局部保证一致性） |
| 效率 | 高（全自动） | 中等（需要手动PS） |
| 返工概率 | 较高 | 较低 |

#### 1.4.2 模式选择建议

| 产品特征 | 推荐模式 |
|----------|----------|
| 有精确数量的可数细节（孔眼、按钮等） | 混合模式 |
| 结构方式容易被AI替换（如grommet vs rod pocket） | 混合模式 |
| 简单结构、无复杂细节 | AI生图模式 |
| 纯色无配件的产品 | AI生图模式 |
| 品牌Logo必须精确 | 混合模式 |
| 场景图为主、产品细节不重要 | AI生图模式 |

#### 1.4.3 展示给用户的选择提示

```
请选择生图精度模式：

1. AI生图模式：适合简单产品，AI全权生成
   - 优点：全自动，效率高
   - 缺点：可数细节（孔眼/按钮数量等）可能偏差，需质检返工

2. 混合模式：适合高精度需求，AI生成背景+用户PS贴入真实产品局部
   - 优点：精度高，真实产品局部保证一致性
   - 缺点：需要手动PS合成

请回复"1"选择AI生图模式，或回复"2"选择混合模式。
（如不确定，可回复产品类型，由 Skill 给出建议）
```

> 注意：精度模式的选择不影响产品特征摘要表和图片规划摘要表的内容，只影响 Phase 2 的执行方式。
> 详见 `references/product_fidelity_lock.md` 第6节。

### 2.2 Prompt 生成规则

#### 2.2.1 提示词结构（四层 + 产品锁）

每张图的 Prompt 必须区分以下层次：

1. **产品不可变层**：真实产品形状、颜色、图案、数量、结构、比例
   - 这部分内容直接来自 `approved_image_plan.json` 中的 `immutable_layer`
   - 是 Prompt 中最优先、最不可改动的部分
   - 例：`a matte black rectangular folding storage rack, single unit, steel material`

2. **产品锁段落**：从保真锁定卡生成的可量化细节约束
   - 这部分内容直接来自 `approved_image_plan.json` 中的 `fidelity_lock.product_lock_paragraph`
   - 放在产品不可变层之后，视觉表达层之前
   - 每张图的 Prompt 必须原样包含此段落，不得修改
   - 模板见 `references/product_fidelity_lock.md` 第5节
   - 示例：
     ```
     【产品锁 — 不可更改，每张图必须包含】
     - EXACTLY 8 grommets per panel, total 16 grommets (2 panels)
       (NOT 6, NOT 7, NOT 9, NOT 10 grommets)
     - GROMMET TOP ONLY
       (NOT rod pocket, NOT tab top, NOT back tab)
     - 108 inches long, reaches floor
       (NOT dragging, NOT too short)
     - Silver stainless steel grommets, brushed satin finish
       (NOT mirror polished, NOT chrome)
     - Pure white fabric
       (NOT cream, NOT ivory)
     【产品锁 — 结束】
     ```

3. **视觉表达层**：场景、构图、信息层级、文字标注
   - 这部分内容来自 `visual_layer`
   - 可以根据创意调整，但不得违反产品不可变层和产品锁段落
   - 例：`placed on a clean white surface, 45-degree front angle, centered composition, soft studio lighting`

4. **合规约束层**：禁用表达、negative prompt
   - 这部分内容来自 `compliance_layer`
   - 确保 Prompt 中不包含敏感词和违规表达
   - 例：`negative prompt: no text overlay, no watermark, no brand logo, no extra products`

#### 2.2.2 Prompt 组装顺序

```
[产品不可变层] + [产品锁段落] + [视觉表达层] + [合规约束层] + [negative prompt]
```

> 注意：产品锁段落是 AI 生图精度的核心约束。它将保真锁定卡中的可量化细节转化为 Prompt 可识别的约束语句，
> 包括精确数量（及排除的邻近错误数量）、精确结构（及排除的错误结构）、尺寸比例、材质光泽和颜色。
> 详见 `references/product_fidelity_lock.md` 第5节。

#### 2.2.3 文字标注规范

- 英文字符大小至少 36px，可以更大
- 优先使用简单数字和短词（如 "6-tier", "304 steel"）
- 避免长句（AI 生成文字容易拼错）
- 文字标注内容必须经过合规检查（对照 `references/claims_compliance.md` 中的115个敏感词清单）

### 2.3 根据精度模式执行

> Phase 2 的执行方式根据用户在 Phase 1 选择的精度模式（`approved_image_plan.json` 中的 `precision_mode`）而有所不同。

#### 2.3.1 AI生图模式（`ai_generation`）

适合简单产品，AI 全权生成。

```
步骤1：读取 approved_image_plan.json
  ↓
步骤2：对每张已确认的图片，按以下顺序生成：
  2a. 组装 Prompt（产品不可变层 + 产品锁段落 + 视觉表达层 + 合规约束层 + negative prompt）
  2b. 加载参考真实产品图
  2c. 调用图片生成 API
  2d. 保存到 /workspace/outputs/[产品ID]/[filename]
  ↓
步骤3：每张图生成后立即进入质检流程（2.5）
  ↓
步骤4：质检通过 → 标记为 PASS
  质检不通过 → 进入返工机制（2.6）
  返工3次仍不通过 → 启动诚实降级协议（2.7）
```

#### 2.3.2 混合模式（`hybrid`）

> 注意：本节与 references/product_fidelity_lock.md 第6.3节内容一致，修改时需同步更新。

适合高精度需求，AI 生成背景 + 用户 PS 贴入真实产品局部。

```
步骤1：读取 approved_image_plan.json
  ↓
步骤2：对每张已确认的图片，AI 生成场景背景（留白）：
  2a. 组装 Prompt（产品不可变层 + 产品锁段落 + 视觉表达层 + 合规约束层 + negative prompt）
      → Prompt 中标注："leave blank space for product overlay"
  2b. AI 生成带留白的场景图（产品关键细节区域留白）
  2c. 保存场景背景图到 /workspace/outputs/[产品ID]/bg_[filename]
  ↓
步骤3：输出合成指引（hybrid_guide_[image_id].md）：
  3a. 指明哪个区域需要贴入真实产品局部
  3b. 提供底图中对应区域的截取建议
  3c. 提供PS合成参数（如缩放比例、旋转角度、光照匹配）
  → 指引模板见 references/product_fidelity_lock.md 第6.4节
  ↓
步骤4：用户在PS中合成：
  4a. 从底图截取真实产品局部
  4b. 贴入AI生成的场景图中
  4c. 调整光照、阴影、比例
  4d. 保存合成后图片到 /workspace/outputs/[产品ID]/[filename]
  ↓
步骤5：合成后重新质检（2.5）
  → 质检时额外检查合成区域与背景的融合度
  → 质检时额外检查贴入的真实产品局部与底图完全一致
  ↓
步骤6：质检通过 → 标记为 PASS
  质检不通过 → 进入返工机制（2.6）
  返工3次仍不通过 → 启动诚实降级协议（2.7）
```

#### 2.3.3 精度模式切换

- 在执行过程中，如用户希望切换精度模式（如从 AI生图模式 切换到 混合模式），需：
  1. 更新 `approved_image_plan.json` 中的 `precision_mode` 字段
  2. 对尚未生成的图片按新模式重新执行
  3. 已生成且质检通过的图片不需要重新生成（除非用户要求）

  质检不通过 → 进入返工机制（2.6）

##### A+ 图片尺寸规范

| 类型 | type 字段值 | 尺寸（像素） | 宽高比 | 适用场景 |

### 2.5 质检流程

> 质检以已锁定的产品事实和保真锁定卡为基准，不得因图片"好看"而放宽事实要求。
> 质检标准详见 `references/qc_checklist.md`。

#### 2.5.1 质检三级状态

| 状态 | 含义 | 后续 |
|------|------|------|
| PASS | 产品事实、保真锁定卡、合规和技术检查全部通过 | 可交付 |
| CONDITIONAL | 有可修复问题 | 修复后重新质检 |
| FAIL | 关键错误（品牌、数量、结构、保真细节、claim 等） | 禁止交付 |

#### 2.5.2 质检检查项

**1. 产品事实检查**
- 生成图中的产品形状是否与真实产品图一致
- 生成图中的产品颜色是否与锁定颜色一致
- 生成图中的产品数量是否与锁定数量一致
- 生成图中的产品结构是否与锁定结构一致
- 不一致 → FAIL

**2. 保真检查（对照保真锁定卡）**
> 除了产品事实检查，还必须对照保真锁定卡检查可量化细节。
> 详见 `references/qc_checklist.md` 第3.5节。

- **可数细节检查**：
  - 逐一清点生成图中的可数部件（孔眼、按钮、口袋等）
  - 与保真锁定卡中的"每单位数量"和"总数"对比
  - 多1个或少1个都是 FAIL
  - 示例：锁定卡要求 8 grommets per panel, total 16，图中必须恰好 16 个

- **结构方式检查**：
  - 检查悬挂/开合/安装方式与保真锁定卡一致
  - 确认未出现被 NOT 排除的错误结构
  - 出现替代结构 = FAIL
  - 示例：锁定卡要求 grommet top，图中不得出现 rod pocket

- **尺寸比例检查**：
  - 生成图中产品的长宽比与底图/锁定卡一致
  - 未出现明显变长/变短/变宽/变窄
  - 示例：锁定卡要求 height:width ≈ 3:1，图中不得变为 2:1 或 4:1

- **材质光泽检查**：
  - 金属表面光泽类型与保真锁定卡一致（brushed satin / mirror polished / matte）
  - 颜色与保真锁定卡一致，无近似色偏差
  - 示例：锁定卡要求 brushed satin，图中不得出现 mirror polished

- **混合模式额外检查**（如使用混合模式）：
  - 贴入的真实产品局部与底图完全一致
  - 合成区域与背景融合自然（光照方向一致、阴影合理、无贴图痕迹）

**3. 合规检查**
- 图片中文字是否包含敏感词（对照 `references/claims_compliance.md` 中的115个敏感词清单）
- 是否有未经验证的性能/安全声明
- 主图背景是否为纯白
- 违规 → FAIL

**4. 技术检查**
- 图片尺寸是否正确
- 图片清晰度是否达标
- 是否有明显 AI 瑕疵（如多余手指、变形文字）
- 有瑕疵 → CONDITIONAL

**5. 文字标注检查**
- 英文字符大小是否至少 36px
- 文字是否有拼写错误
- 文字内容是否与计划一致
- 有问题 → CONDITIONAL

### 2.6 返工机制

#### 2.6.1 返工触发条件

- 质检状态为 CONDITIONAL → 自动返工
- 质检状态为 FAIL → 停止，分析原因后返工

#### 2.6.2 返工流程

```
步骤1：分析质检不通过的原因
  ↓
步骤2：根据原因调整 Prompt
  - 产品事实问题 → 检查 immutable_layer 是否正确传入，强化描述
  - 保真问题 → 检查产品锁段落是否正确传入，强化可量化细节约束
  - 合规问题 → 替换违规表达
  - 技术问题 → 调整生成参数
  - 文字问题 → 简化文字或增大字号
  ↓
步骤3：重新生成图片
  ↓
步骤4：重新质检
  ↓
步骤5：最多返工 3 次
  - 3 次后仍不通过 → 启动诚实降级协议（2.7）
```

#### 2.6.3 返工限制

1. 返工只能调整视觉表达层和合规约束层，不得修改产品不可变层和产品锁段落。
2. 返工次数上限为 3 次，超过后启动诚实降级协议。
3. 每次返工都需记录原因和调整内容。

### 2.7 诚实降级协议

> 注意：本节与 references/product_fidelity_lock.md 第7节内容一致，修改时需同步更新。

> 当 AI 生图返工 3 次后，仍无法通过保真质检时，启动诚实降级协议。
> 详见 `references/product_fidelity_lock.md` 第7节。

#### 2.7.1 触发条件

```
返工第1次：调整Prompt → 重新生成 → 质检
  ↓ （不通过）
返工第2次：加强约束词 → 重新生成 → 质检
  ↓ （不通过）
返工第3次：最后一次尝试 → 重新生成 → 质检
  ↓ （仍不通过）
启动诚实降级协议
```

#### 2.7.2 降级流程

1. **标记 CONDITIONAL**
   - 将该图片的质检状态标记为 CONDITIONAL
   - 不得标记为 PASS，不得假装通过了质检

2. **记录具体偏差**
   - 哪些细节与保真锁定卡不一致
   - 偏差程度（轻微/中等/严重）
   - 建议的修复方式

3. **告知用户**
   - 展示生成图与底图的对比
   - 列出具体偏差项
   - 建议手动修正方式（PS贴图/局部重绘）
   - 或建议切换到混合模式

4. **等待用户决策**
   - 用户手动修正 → 修正后重新质检
   - 用户切换混合模式 → 按混合模式重新生成（2.3.2）
   - 用户接受偏差 → 标注"已知情接受"，记录在QA报告中

#### 2.7.3 降级报告模板

```markdown
## 降级报告 - {image_id}

### 返工历史

## 确认门禁机制

> 确认门禁是 Phase 1 和 Phase 2 之间的"关卡"。
> 用户没有回复确认前，不得生成最终 Prompt 或图片。

### 3.1 门禁文案

在 Phase 1 的4项内容（产品特征摘要表、图片规划摘要表、精度模式选择、A+ 内容规划摘要表）全部展示完毕后，必须显示以下门禁文案：

```
========================================
以上为 Phase 1 规划方案。请确认后我再生成详细 Prompt 和图片。

可选操作：
- 回复"确认"：仅执行 Listing 图
- 回复"确认+A+"：执行 Listing 图 + A+ 10张图
- 回复"确认A+"：仅执行 A+ 10张图
- 回复"确认图1-5"：部分执行
- 回复修改意见：调整后重新确认

注意事项：
- 请确认产品特征摘要表和保真锁定卡是否准确
- 请选择精度模式（AI生图模式 / 混合模式）
- 请确认是否生成 A+ 内容（桌面版5张 + 移动版5张）
- 阻断项未解决前，部分图片可能无法生成
========================================
```

### 3.2 部分确认逻辑

#### 3.2.1 用户回复"确认图X-Y"

当用户只想确认部分图片时：
1. 解析用户指定的图片序号范围（如图1-5 表示 main-01 + listing-01 到 listing-04）。
2. 只将确认的图片加入 `approved_image_plan.json` 的 `approval_info.approved_images`。
3. `approval_status` 设为 `"partial"`。
4. 未确认的图片不生成 Prompt 和图片，但保留在计划中供后续确认。

#### 3.2.2 部分确认的后续

- 用户可以在后续对话中追加确认其他图片。
- 每次追加确认都需更新 `approved_image_plan.json`。
- 已确认并生成的图片不需要重新生成（除非用户要求修改）。

### 3.3 修改后重新确认

#### 3.3.1 用户回复修改意见

当用户对规划方案提出修改意见时：
1. 记录用户的修改意见到 `approval_info.modification_notes`。
2. 根据修改意见调整聊天中展示的对应内容（产品特征摘要表 / 图片规划摘要表 / 精度模式选择）。
3. 修改完成后，重新显示门禁文案，等待用户再次确认。

#### 3.3.2 修改限制

1. 修改不得违反产品事实和保真锁定卡。
   - 如果用户的修改意见涉及改变产品事实或保真锁定卡 → 告警并要求用户提供新证据，重新锁定。
2. 修改不得引入合规风险。
   - 如果用户的修改意见包含敏感词 → 告警并提供替代表达。
3. 修改后的内容必须重新通过合规检查（对照 `references/claims_compliance.md`）。

#### 3.3.3 修改后确认流程

```
用户提出修改意见
  ↓
记录修改意见
  ↓
判断修改是否涉及产品事实/保真锁定卡变更
  ├─ 是 → 告警，要求用户提供新证据，重新锁定
  └─ 否 → 继续
  ↓
判断修改是否引入合规风险
  ├─ 是 → 告警，提供替代表达
  └─ 否 → 继续
  ↓
更新聊天中展示的对应内容
  ↓
重新通过合规检查
  ↓
重新显示门禁文案
  ↓
等待用户确认
```

### 3.4 确认门禁的硬性规则

1. **不得跳过门禁**：即使用户说"快点生成"，也必须先显示门禁文案并获得明确确认。
2. **确认必须明确**：用户必须明确回复"确认"或"确认图X-Y"，模糊回复（如"好的""可以"）需进一步确认。
3. **门禁只认文字**：确认以用户文字回复为准，不自动推断用户意图。
4. **阻断项优先**：如果存在阻断项，即使用户确认，阻断项相关的图片也不得生成，需先解决阻断项。
5. **精度模式必须选择**：用户确认时必须同时选择精度模式（AI生图模式 / 混合模式），否则需进一步确认。
