# Amazon Image Rules / 亚马逊图片规范

> 本文件是 Amazon Visual Designer 的图片规范基线。
> 所有主图、副图、A+ 内容图片的生成与质检，必须以本文件为准。
> 实际上传前，以当前 Seller Central 类目要求为准（类目间存在差异）。
> 本规范与 `claims_compliance.md`（合规与敏感词）联动执行。

## 1. 主图要求

> 主图是 Listing 的"门面"，也是亚马逊审核最严格的图片。
> 主图必须基于真实产品白底图，不得凭空生成（参见 SKILL.md Hard Rule 3）。

### 1.1 背景要求

- **纯白背景**：RGB 值为 (255, 255, 255)，即纯白。
- 不允许任何渐变、阴影底色、场景背景。
- 产品投影须在纯白底上自然呈现，不得人为添加色块。

> 检查方法：取背景区域像素，确认 R、G、B 三通道均为 255。

### 1.2 产品占比

- **产品须占图片面积的 85% 以上**。
- 产品居中，四周留白均匀且最小化。
- 不得有大面积空白导致产品过小。

> 检查方法：计算产品主体 bounding box 面积 / 图片总面积 ≥ 0.85。

### 1.3 画面内容

- **无水印**：不得添加任何水印。
- **无文字**：主图不得包含任何文字（部分类目除外，见 1.4）。
- **无 Logo**：不得在主图中叠加品牌 Logo（产品本身自带的 Logo 除外）。
- **无多余道具**：不得添加用于装饰的道具、配饰、背景物。仅展示产品本身。
- **无人体**：主图一般不出现人体（服装、佩戴类等特殊类目除外）。

### 1.4 类目例外

> 部分类目允许主图包含少量文字或特定元素，但**必须以当前 Seller Central 类目规则为准**。

| 类目 | 可能的例外 | 备注 |
|------|-----------|------|
| 图书 / 媒体 | 可展示封面文字 | 仅限产品本身自带的文字 |
| 服装 | 可使用模特图 | 须为真实穿着场景 |
| 有包装的产品 | 可展示包装 | 包装上的文字属产品自带 |

> 例外不等于豁免：即便类目允许，仍须遵守合规与敏感词规则。

### 1.5 主图保真要求（项目硬规则）

- 主图必须基于真实产品白底图生成或处理。
- 缺少真实产品图时，**不得生成主图**（参见 SKILL.md Hard Rule 3）。
- 不得凭空生成不存在的产品、颜色、图案、配件。
- 主图中的产品形状、颜色、图案、数量、结构必须与 `approved_image_plan.json` 中的 `product_truth` 锁定的事实完全一致。

## 2. 副图要求

> 副图用于补充主图无法展示的信息，帮助买家决策。
> 副图可包含文字标注，但仍须遵守合规规则。

### 2.1 副图类型与目的

> 核心原则：**每张副图只服务一个明确目的**，不堆砌信息。

| 副图类型 | 目的 | 示例 |
|----------|------|------|
| 不同角度图 | 展示主图未呈现的面 | 侧面、背面、底部、俯视 |
| 使用场景图 | 展示产品在真实环境中的使用 | 厨房、客厅、办公桌等场景 |
| 功能细节图 | 放大展示关键功能部件 | 接口、卡扣、调节机构等特写 |
| 尺寸信息图 | 用图形化方式标注尺寸 | 带标注线的产品三视图 |
| 包装内容图 | 展示包装内含物清单 | 所有配件平铺展示 |
| 对比图 | 展示不同型号/规格差异 | 多颜色、多尺寸并列 |
| 安装/使用步骤图 | 分步展示使用方法 | 1-2-3 步骤示意 |

### 2.2 副图内容规范

- **每张图一个明确目的**：不混合多个主题，避免信息过载。
- **可包含文字标注**：副图允许文字，但须遵守第 5 节文字标注规范和 `claims_compliance.md`。
- **背景灵活**：副图背景不限于纯白，可使用场景、渐变、色块，但不得喧宾夺主。
- **产品保真**：产品本身的形状、颜色、图案、数量、结构必须与主图及产品事实一致。
- **真实性**：使用场景须真实合理，不得虚构使用场景或效果。

### 2.3 副图排序建议

> 副图顺序影响买家浏览体验，建议按以下逻辑排序：

1. **第 1 张副图（Listing 第 2 图）**：次重要角度或核心卖点场景。
2. **第 2-3 张**：功能细节、使用场景。
3. **第 4-5 张**：尺寸信息、包装内容。
4. **第 6 张及以后**：对比、安装步骤、附加信息。

## 3. 尺寸规范

> 尺寸规范分 Listing 图与 A+ 内容图两类。
> 项目默认尺寸见 SKILL.md，本节补充完整规范。

### 3.1 Listing 图（主图与副图）

| 项目 | 要求 | 说明 |
|------|------|------|
| 最小尺寸 | 500 × 500 px | 低于此尺寸无法上架 |
| 推荐尺寸 | 1600 × 1600 px 及以上 | 支持缩放功能，提升转化 |
| 长边最大 | 10000 px | 超出可能被压缩 |
| 长宽比 | 1:1 或竖版 | 正方形最通用；竖版适用于服装等类目 |
| 项目默认 | 1600 × 1600 px（方形 1:1） | 见 SKILL.md，可按需调整 |
| 文件格式 | JPEG / TIFF / PNG / GIF | 主图推荐 JPEG |
| 文件大小 | 单张不超过 10 MB | 建议控制在 5 MB 以内以保证加载速度 |

> 缩放功能要求：图片长边 ≥ 1000 px 时，买家可点击放大查看细节。推荐 1600 px 以上。

### 3.2 A+ 内容图

> A+ 内容是亚马逊品牌注册卖家可使用的高级图文描述模块，分为桌面版与移动版。
> 移动版必须独立构图，不可直接缩放桌面版（参见 SKILL.md Hard Rule 9）。
> 详见 `references/aplus_content_guide.md`。

#### 3.2.1 尺寸与比例

| 资产类型 | 尺寸 | 比例 | 构图方向 | 说明 |
|----------|------|------|----------|------|
| A+ 桌面版 | 1464 × 600 px 或 1940 × 1200 px | 横版（约 2.44:1 或约 1.62:1） | 横向左右分区 | 宽幅横图，适合 PC 端浏览 |
| A+ 移动版 | 1600 × 1200 px | 竖版（4:3） | 纵向上下分区 | 必须独立构图，适合手机竖屏 |

#### 3.2.2 构图差异

| 维度 | 桌面版 | 移动版 |
|------|--------|--------|
| 构图方向 | 横向，左右分区 | 纵向，上下分区 |
| 产品位置 | 左侧 | 上方居中 |
| 文字位置 | 右侧 | 下方 |
| 信息密度 | 可较高 | 必须精简 |
| 文字字号 | ≥ 36px | ≥ 48px |
| 场景宽度 | 宽幅全景 | 竖向聚焦 |

#### 3.2.3 A+ 内容模块类型

A+ 内容通常包含以下模块类型（每个模块对应一张图片）：

| 模块类型 | 说明 | Desktop | Mobile |
|----------|------|---------|--------|
| 品牌横幅 | Hero 场景，品牌形象展示 | 宽幅场景横图 | 纵向产品大图 |
| 卖点展示 | 核心卖点图文说明 | 左右并列 | 上下排列 |
| 规格信息 | 产品规格信息图卡 | 横向多列布局 | 纵向单列布局 |
| 场景应用 | 产品使用场景展示 | 宽幅场景 | 竖向场景 |
| 套装内容 | 包装内容平铺展示 | 左右展示 | 纵向居中展示 |

#### 3.2.4 A+ 文字标注规范

- 桌面版文字字号 ≥ 36px（与 Listing 副图一致）
- 移动版文字字号 ≥ 48px（比桌面版更大，确保小屏可读）
- 移动版文字以短词和数字为主，避免长句
- 所有文字标注须通过 `claims_compliance.md` 敏感词扫描
- A+ 图片文字量通常多于 Listing 图，拼写错误风险更高，建议后期 PS 叠加文字层

#### 3.2.5 A+ 内容禁止事项

- **禁止缩放**：移动版不得直接缩放桌面版图片（Hard Rule 9）
- **禁止敏感词**：与 Listing 图一致，不得出现 115 个敏感词
- **禁止竞品**：不得出现其他品牌的产品或 Logo
- **禁止虚假声明**：所有功能声明须有证据支持
- **禁止水印**：不得包含任何水印（包括 AI 生成水印）
- **禁止边框**：不得添加装饰性边框（A+ 模板自带边框除外）

### 3.3 分辨率与色彩

- **分辨率**：72 DPI（网页标准），无需打印分辨率。
- **色彩空间**：sRGB，确保浏览器显示一致。
- **色彩还原**：产品颜色须与真实产品一致，不得过度修图导致色差。

## 4. 禁止事项

> 以下行为一律禁止，违反者质检状态为 FAIL（参见 SKILL.md QC Quality Gate）。

### 4.1 内容真实性禁止

- **不得误导性展示**：不得展示产品不具备的功能、效果或性能。
- **不得添加不存在配件**：图片中出现的所有配件、组件必须与产品事实一致，不得虚构。
- **不得夸大效果**：不得通过修图、合成夸大产品效果（如过度提亮、虚假对比）。
- **不得虚构场景**：使用场景须真实合理，不得虚构与产品无关的场景误导买家。

### 4.2 文案与声明禁止

- **不得使用绝对化用语**：如"100%""best""No.1""绝对""最强"等（联动 `claims_compliance.md`）。
- **不得使用敏感词**：参见 `claims_compliance.md` 第 1 节 115 项敏感词清单。
- **不得使用未授权的功能声明**：所有功能声明须有 A 级证据（参见 `claims_compliance.md` 第 4 节）。
- **不得展示虚假认证**：未持有有效证书不得展示认证标志或声明。

### 4.3 竞品与版权禁止

- **不得展示竞品**：图片中不得出现其他品牌的产品、Logo 或可识别的竞品包装。
- **不得贬低竞品**：不得通过对比图贬低其他品牌。
- **不得使用未授权素材**：所有图片素材须有合法使用权，不得盗用他人图片。
- **不得侵犯肖像权**：使用模特图须有肖像授权。

### 4.4 技术性禁止

- **不得添加边框**：图片不得添加装饰性边框（部分 A+ 模板除外）。
- **不得添加水印**：包括品牌水印、版权水印等（副图亦同）。
- **不得使用动画**：Listing 图片不得使用 GIF 动画（主图严格禁止）。
- **不得拼接多产品**：主图不得拼接多个不同产品（套装产品除外，须明确标注）。

## 5. 文字标注规范

> 副图和 A+ 内容可包含文字标注，须遵守以下规范（联动 SKILL.md Image Generation Rules）。

### 5.1 字号要求

- **英文字符最小 36 px**，可以更大。
- 移动端 A+ 图片中的文字建议更大，确保小屏可读。
- 避免长句，AI 生成文字容易拼错。

### 5.2 内容要求

- **优先使用简单数字和短词**：如"3 in 1""BPA-Free（须有证据）""Easy Clean"。
- **避免长句**：单行不超过 5 个英文单词。
- **避免复杂语法**：不用从句、被动语态等复杂结构。
- **标注须与产品一致**：文字描述的功能、参数必须与产品事实一致。

### 5.3 合规要求

- 所有文字标注须通过 `claims_compliance.md` 敏感词扫描。
- 功能声明须有 A 级证据。
- 不得使用绝对化用语。
- 认证声明须标注具体机构与编号。

## 6. 质检对接

> 本文件规范在 QC 质检阶段被调用，质检状态分三级（参见 SKILL.md QC Quality Gate）。

### 6.1 质检检查项映射

| 检查项 | 对应章节 | FAIL 触发条件 |
|--------|----------|---------------|
| 主图背景 | 1.1 | 背景非纯白（RGB ≠ 255,255,255） |
| 主图占比 | 1.2 | 产品占比 < 85% |
| 主图内容 | 1.3 | 含水印/文字/Logo/多余道具 |
| 主图保真 | 1.5 | 产品与产品事实不一致 |
| 副图目的 | 2.1 | 单张图混合多个目的 |
| 尺寸规范 | 3.1 / 3.2 | 尺寸不达标或比例错误 |
| 误导展示 | 4.1 | 展示不具备的功能/配件 |
| 绝对化用语 | 4.2 | 出现"100%""best"等 |
| 敏感词 | 4.2 | 出现 115 项敏感词中的任一 |
| 竞品展示 | 4.3 | 出现其他品牌产品/Logo |

### 6.2 质检流程

1. **主图质检**：逐项检查第 1 节，任一不达标 → FAIL。
2. **副图质检**：检查第 2、5 节，关键项不达标 → FAIL，次要项 → CONDITIONAL。
3. **合规扫描**：对图片中所有文字执行敏感词扫描（联动 `claims_compliance.md`）。
4. **尺寸校验**：核对第 3 节尺寸规范。
5. **输出报告**：写入 `qa_report.md`，记录每张图的检查结果与状态。

### 6.3 与其他规则文件的关系

| 关联文件 | 关系 |
|----------|------|
| `claims_compliance.md` | 图片文字标注的合规检查依据 |
| `prompt_template.md` | 图片生成 Prompt 的三层结构引用本文件 |
| `qc_checklist.md` | QC 质检清单调用本文件执行图片规范检查 |
| `product_truth_rules.md` | 产品事实基线，用于校验图片中的产品保真 |

## 7. 底图最低要求

> 本节定义用户提供的原始产品底图（白底图/实拍图）的最低要求。
> 底图是产品保真的核心证据，底图不足会直接导致生成图与真实产品出现细节偏差。
> 详见 `references/workflow.md` 第 0.1.3 节底图覆盖度评估。

### 7.1 底图数量与覆盖度

| 评估维度 | 最低要求 | 不足时处理 |
|----------|----------|-----------|
| 底图数量 | 至少 3 张不同角度/细节 | 不足 3 张时告警，要求补充 |
| 正面视角 | 至少 1 张清晰正面图 | 缺失则阻断流程 |
| 侧面/背面视角 | 至少 1 张侧面或背面图 | 缺失则告警，用户可确认继续 |
| 关键细节特写 | 至少 1 张关键细节图 | 缺失则告警，用户可确认继续 |
| 清晰度 | 所有图片清晰可辨 | 模糊则阻断流程 |

### 7.2 底图质量标准

- **分辨率**：建议每张底图分辨率不低于 1000 × 1000 px
- **背景**：首选纯白背景（#FFFFFF），纯色背景可接受
- **光线**：光线均匀，无明显过曝或欠曝
- **完整性**：产品主体完整可见，不被遮挡
- **色彩还原**：颜色准确，无明显偏色

### 7.3 底图不足时的风险提示

当底图覆盖度不足时，Skill 必须告知用户以下风险：

1. **场景图偏差**：缺少侧面/背面视角时，场景图中的产品可能呈现错误结构
2. **细节图失真**：缺少关键细节特写时，细节图中的可数部件（孔眼、按钮等）可能数量错误
3. **材质误判**：底图清晰度不足时，材质光泽和纹理可能被错误识别
4. **返工成本**：底图不足导致生成图偏差的返工成本，远高于补充底图的成本

> 关键原则：宁可让用户补图，也不要在信息不足的情况下盲目生成。
